# Critique reports

Output of the experimental critique pass (`kb/instructions/critique-note.md`): an on-demand, divergent review that builds the strongest case against a note's central commitment and checks whether the note already answers it.

Reports here are **gitignored** — they are regenerable working artifacts, read once and acted on, like connect reports. Only this README is tracked, to keep the directory in git.

If critique later becomes durable standing context co-loaded at review time, un-ignore the directory in `.gitignore`; that change is the commitment to durability.

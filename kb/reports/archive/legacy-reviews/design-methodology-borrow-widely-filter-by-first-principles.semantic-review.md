WARN:
- [Completeness] The adoption filter implicitly presents a taxonomy of source categories with distinct adoption paths: (1) first-principles derivation, (2) programming fast pass, (3) legal drafting (untested hypothesis), (4) everything else, (5) empirical observation. But category 4 ("Everything else earns its way in... these get adopted when first-principles reasoning supports them") collapses into category 1 — first-principles derivation applied to non-programming domains. The cognitive science example in category 4 (three-space memory "because it maps to a real architectural need") is exactly first-principles reasoning applied to a borrowed pattern. The framework's real structure has three distinct adoption mechanisms: first-principles derivation (the main filter, which also covers "everything else"), the programming fast pass (reduced evidential bar), and empirical observation (bypasses the filter entirely). Presenting five subsections with parallel headings overstates the framework's dimensionality. Legal drafting and "everything else" are not distinct tiers; they are applications of the same filter at different maturity stages.

- [Completeness] The note enumerates four first principles: "finite context windows, no import/resolution mechanism, agents reason over text, everything loaded must compete for attention." But the asymmetry argument deploys a fifth: "no persistent state between sessions." This fifth principle does critical work — it is the wedge that separates LLM agents from human cognition ("Human cognition is associative, embodied, affective. LLM agents process text in a fixed-size window with no persistent state between sessions. The mechanisms are different enough that cognitive science analogies need independent justification"). If statelessness is a constraint that "won't change" and grounds a central design conclusion, its absence from the enumerated set is a gap. A reader relying on the enumerated list as "the constraints of the domain" would miss a principle the note itself treats as load-bearing. Additionally, "everything loaded must compete for attention" may itself decompose into two separable constraints: finite capacity (hard token limit) and soft degradation (attention quality drops before the limit is reached). Whether these should be distinct principles is arguable, but the note treats them as one.

- [Grounding — domain coverage] The note claims "Convergence across independent projects is stronger evidence than any single design argument" and cites Thalo as the sole instance. The Thalo note confirms the factual claims — Tree-Sitter grammar, typed entities, 27 validation rules, independent development. But the convergence evidence supports only that programming patterns transfer to knowledge systems. It does not support the broader claim the note uses it for: that programming patterns deserve a methodological fast pass *relative to other source disciplines*. Thalo did not evaluate cognitive science patterns and reject them; it simply never tried them. The Thalo note itself observes: "They don't have a learning theory." The convergence evidence validates the positive half of the asymmetry (programming transfers) but is silent on the negative half (other sources should not get the same treatment). The note presents this evidence in the context of the full asymmetry argument without distinguishing what it covers from what it does not.

INFO:
- [Completeness] Boundary case: formal/mathematical reasoning as a source discipline. The fast-pass criterion is "bounded processors composing text under constraints," and the asymmetry argument says programming systems are "formal, compositional, text-based." Mathematics is formal and compositional but not text-based in the same operational sense — category theory, information theory, and graph theory provide analytical frameworks rather than operational patterns. The note's framework does not clearly route mathematics: it shares the "formal and compositional" properties that justify the programming fast pass but lacks the "text-based interpreter" mechanism the bet rests on. Mathematics would likely fall under "everything else earns its way in" (first-principles support needed), but its formal-and-compositional nature makes it a boundary case between the fast pass and the general filter.

- [Completeness] Boundary case: what happens when first principles conflict? "Agents reason over text" favors explicit, verbose context. "Everything loaded must compete for attention" favors compression. These pull in opposite directions for any given design decision. The note provides a filter for adopting ideas (derive from first principles) but no method for resolving tensions between the principles themselves. This is an edge of the framework — a methodology note that says "derive from constraints" without addressing what to do when constraints oppose each other.

- [Completeness] Boundary case: the simplest possible instance of "borrowing." If an idea from cognitive science happens to also be derivable from first principles (e.g., "chunking information improves processing" could be derived from "everything loaded competes for attention"), does it enter through the first-principles path or the "everything else" path? The note says cognitive science patterns "get adopted when first-principles reasoning supports them," which suggests first-principles derivation is what matters and the source discipline is irrelevant. But the asymmetry section frames the question as being about source disciplines. The framework conflates two different questions: "where did the idea come from?" (source routing) and "how is it justified?" (evidential support). The adoption filter answers the second, but the presentation — organized by source discipline — frames it as answering the first.

- [Grounding] The note states legal drafting is "untested" — "we haven't yet borrowed a concrete technique from law and applied it successfully." The linked legal-drafting note documents a detailed table of six legal techniques with context engineering analogues, rated by mapping tightness (tight/moderate/loose), plus a case study (ABC contracts). The methodology note draws a distinction between "identified a conceptual mapping" and "borrowed a technique and validated it in practice," but this distinction is implicit. The legal-drafting note's detailed mappings (defined terms as glossaries, structural conventions as templates, enumeration as schemas — all rated "tight") suggest the source is more developed than "untested" conveys, even if no technique has completed a full borrow-apply-validate cycle. The word "untested" risks underselling work already done.

- [Internal consistency] The note says "Empirical observation is the second strongest source — but weaker than first principles" and ranks it in an ordered hierarchy. Two paragraphs later, it says observation "doesn't go through the borrowing/adoption filter at all. It's evidence from this system, not transferred from another domain." Ranking something "second strongest" within a hierarchy (implying it belongs to the hierarchy) and then saying it bypasses the hierarchy entirely creates a tension. The note partially resolves this with the quantity-vs-weight distinction and by distinguishing "source strength" from "filter membership," but the initial ranking framing invites the reader to place observation on the same scale as first-principles and programming-pattern borrowing, which the subsequent text explicitly denies. The intended meaning is recoverable but the framing is slightly misleading.

The wikiwiki principle reference ("capture observations freely, then refine") is coherent with this framing — observations are cheap to capture and need repetition before they carry confidence.

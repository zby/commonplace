# Phase-1 audit — core authoring/ingest skills in `kb/instructions/`

Group: the seven promoted core skills — `cp-skill-write`, `cp-skill-connect`, `cp-skill-validate`, `cp-skill-convert`, `cp-skill-ingest`, `cp-skill-snapshot-web`, `cp-skill-health-check` (each directory's `SKILL.md` plus auxiliary files). Date: 2026-07-23.

Theory applied: [self-improving-systems cluster head](../../../notes/self-improving-systems-README.md); [behavioral authority](../../../notes/definitions/behavioral-authority.md); [the proposal-selection loop](../../../notes/a-proposal-selection-loop-requires-search-evaluation-and-retention.md) (strict evaluation criterion); [false-positive generation is filtered; false-positive acceptance becomes operative](../../../notes/false-positive-generation-is-filtered-before-retention.md) (automate-search-first); [warranted autonomy is bounded by oracle domain](../../../notes/warranted-autonomy-is-bounded-by-oracle-domain.md); [retrieval failure is reflection failure](../../../notes/retrieval-failure-is-reflection-failure.md); [the declared Commonplace frame](../../../reference/commonplace-declared-frame.md); [two-layer execution system](../../../notes/theory-and-methodology-form-a-two-layer-execution-system.md). The parked [`warn` conditional](../warn-outcome-vs-loop-evaluation.md) is applied where an artifact touches the review/fix pipeline.

Shared authority facts, established once: all seven are promoted skills. `commonplace-init` copies `kb/commonplace/instructions/<name>/` into `.claude/skills/` and `.agents/skills/` for every name in `MANIFEST.promoted_skills` (`src/commonplace/scaffold_manifest.py:82-90`; the copy loop is `src/commonplace/cli/init_project.py:212-222`). In the source repo the runtime directories are symlinks into `kb/instructions/`. The real authority path is therefore: **consumer** = an agent in a forked execution context (`context: fork`, model pinned per skill); **channel** = harness skill loading, triggered by description match or explicit `/cp-skill-*` invocation; **force** = instruction (standing instruction for the fork). A secondary, weaker path exists for all of them: searchable prose in `kb/instructions/` read as reference (advice force). Per the [declared frame](../../../reference/commonplace-declared-frame.md), the skills, the commands they call, and the harness are computational components inside the frame; the directing human is the frame's human component.

A consequence of the copy mechanism, used repeatedly below: only `SKILL.md` is loaded by the harness. Auxiliary files in the directory are copied but consumed **only if the SKILL.md text instructs reading them** — an unreferenced auxiliary file has no consumer, hence no force, regardless of content. Likewise, markdown-relative links inside a promoted SKILL.md resolve against the runtime path, not `kb/instructions/` — which is exactly the warning `kb/instructions/README.md` records ("Do not assume markdown-relative links inside the skill will resolve from the runtime copy").

## kb/instructions/cp-skill-write/SKILL.md

- **Authority:** As above — agent via harness skill loading, instruction force, `model: opus`. Additionally cited by name from `AGENTS.md`'s Skills section and as the terminal handoff of `cp-skill-convert` and the entry point for authoring in general; other skills route work to it, so it is the KB's principal write channel.
- **Loop function:** For the KB-content loop: **search** (the Step 4 duplicate check and in-hand link candidates — deliberately narrow, with full prospecting delegated to `cp-skill-connect`) and **retention** (Step 5 saves the note into the library, where it is immediately operative: discoverable via the description/tag retrieval wires, loadable by any later session). Evaluation is delegated, split by oracle: structural to `commonplace-validate` (Step 6 — reject-capable within the authoring procedure: "Fix structural failures before stopping"), semantic to the directing human and downstream review. Self-referentially, when the artifact being written is itself an instruction, gate, or COLLECTION.md, this skill is the **retention step of the system-change loop** — the exact moment the workshop's authority-path-1 question is about.
- **Oracle warrant:** Respected. The skill treats validation pass as establishing structure only ("Fix structural failures"), never as semantic acceptance; verification (`user-verified`) is explicitly reserved to the human ("Never grant user verification implicitly"). The Step 5 rule removing `user-verified` on substantive edits keeps a stale attestation from surviving as a trusted mark — instruction-force only, but replicated across every skill-based edit channel (write, both revise skills, the review archiver), and the review side has a code consumer (`commonplace-review-target-selector --user-verified`) that treats the mark as a review-scope filter, so a wrongly-preserved mark is at least re-reviewable.
- **Retention:** The commitments the skill retains are mostly loaded at the time they matter: it **hard-fails** on a missing `COLLECTION.md` (Step 2) and refuses type fallback (Step 3) — strong retrieval wires, enumeration-like rather than best-effort. The connect handoff ("The suggestion is not optional polish") is an advisory wire, but the obligation has a downstream detector: the validator's batch orphan signal (`cp-skill-validate` description: "batch signals such as orphans"). One wire is broken — see finding 1.
- **Findings:**
  1. **Runtime-relative link break.** Line 94: "[link-vocabulary.md](../../reference/link-vocabulary.md)". From the runtime copy (`.claude/skills/cp-skill-write/SKILL.md`) this resolves to `<root>/reference/link-vocabulary.md`, which exists in no layout; in an installed KB the target is `kb/commonplace/reference/link-vocabulary.md`. The lineage-labeling guidance is thereby unreachable exactly when the executing agent needs it (recording `Derived into:`/`Operationalized into:` edges — the same edge family this workshop's digest will depend on). This is a represented constraint with a cut wire — [retrieval failure is reflection failure](../../../notes/retrieval-failure-is-reflection-failure.md) — and a direct violation of the promoted-skill path rule in `kb/instructions/README.md`.
  2. (Observation, no change.) Retention precedes semantic evaluation by design: a written note is operative on save, gated only structurally. Under the [declared frame](../../../reference/commonplace-declared-frame.md) the accept gate is allocated to the directing human plus downstream review; the skill neither claims nor exercises unattended semantic acceptance, so [warranted-autonomy](../../../notes/warranted-autonomy-is-bounded-by-oracle-domain.md) is not exceeded. Consistent; worth naming in the future digest as the allocation the write path assumes.
- **Disposition:**
  1. `apply` — replace the markdown-relative link with a layout-explicit prose pointer, e.g.: `…and [link-vocabulary](../../reference/link-vocabulary.md)'s test fits` → `…and the link-vocabulary reference doc's test fits (kb/reference/link-vocabulary.md in this repo; kb/commonplace/reference/link-vocabulary.md in installed KBs)`.
  2. no change.

## kb/instructions/cp-skill-connect/SKILL.md

- **Authority:** Agent via harness skill loading, instruction force, `model: opus`. Also invoked as a subroutine by `cp-skill-ingest` (delegated skill call — a second instruction-force channel). Its *output* has deliberately weak authority: a gitignored report under `kb/reports/connect/`, consumed as working context/advice by a future writer or by the invoking ingest run.
- **Loop function:** Pure **search** for the link-graph loop, and it says so in the theory's own shape: "Connect is the skill that pays the cost of active search." The articulation test, label authorization, and off-authorisation routing are search-side pruning, not evaluation — under the [strict criterion](../../../notes/a-proposal-selection-loop-requires-search-evaluation-and-retention.md) their only downstream effect is which candidates the report surfaces for a future author; no verdict controls select/discard/block/rollback of anything operative. Retention is explicitly refused: "the report is the *only* connection output"; "**Never** edit the target artifact, other artifacts, indexes…". This is the cleanest instance in the group of [automate-search-first](../../../notes/false-positive-generation-is-filtered-before-retention.md): heavy model automation concentrated entirely in the function whose false positives get filtered downstream (a bad candidate dies in the report), with acceptance left to the human/author who commits the edge.
- **Oracle warrant:** Largely n/a (nothing is accepted into the library). The one trusted oracle it consumes: a tag-README's `complete: true` mark, which licenses skipping the by-tag `rg` sweep. That is the sharpest trusted-wire case in [retrieval-failure-is-reflection-failure](../../../notes/retrieval-failure-is-reflection-failure.md) — "the claim cuts the wire precisely where a process was relying on it" — but the trust is warranted here because the mark is validator-enforced, not asserted (`kb/types/tag-readme.md`, ADR 026: enforced-or-omitted). The skill's skip-and-record-the-skip behavior is the correct consumption of an enforced mark.
- **Retention:** Reports are declared regenerable ("can be regenerated from the source artifact plus current KB state") — deliberate non-retention, with cumulativity carried by the library, not the report layer. The `Maintenance Observations` section is a retained-signal channel whose scope is honestly stated ("best-effort signal inside a gitignored, regenerable report… a later explicit maintenance or triage step *may* promote entries; connect does not do that promotion"). No named consumer routinely harvests it from standalone runs (ingest is explicitly told to treat it as non-actionable), so the wire is thin — but the artifact says so, which is what the theory requires of a best-effort channel.
- **Findings:** No change suggested. Positive exhibit for the future digest: a skill whose loop classification, automation placement, and trusted-mark consumption all match the cluster's prescriptions without having been written from them.
- **Disposition:** n/a (no findings).

## kb/instructions/cp-skill-validate/SKILL.md

- **Authority:** Agent via harness skill loading, instruction force. The substantive verdict, however, comes from `commonplace-validate` (code — enforcement-grade oracle, `src/commonplace/lib/validation.py`); the skill's own contribution is scoping ("Prefer the narrowest target"), invocation, and verbatim reporting.
- **Loop function:** **Evaluation** invocation — but where the verdict lands differs by flow, and the difference matters. In the authoring flow (invoked from `cp-skill-write` Step 6) a failure blocks completion of the procedure — a real reject-capable gate inside the write loop. On **incumbents** (maintenance sweeps, `all`), the verdict blocks nothing operative: a failing note keeps full behavioral authority, and the verdict's only consumer is a subsequent fix. That is the structure the parked [`warn` analysis](../warn-outcome-vs-loop-evaluation.md) diagnosed for the review gates, appearing here on the deterministic side.
- **Oracle warrant:** Exemplary scoping, worth quoting: "Report the script output directly. Do not interpret, supplement, or override its findings." That sentence keeps the model from extending the deterministic oracle's verdict beyond its domain, and the final routing line sends judgment-based checks to the review pipeline instead of letting the structural oracle be read as a semantic gate. This is [warranted autonomy](../../../notes/warranted-autonomy-is-bounded-by-oracle-domain.md) practiced: full computational autonomy exactly where the oracle (deterministic validation) warrants it, and an explicit handoff at the domain boundary.
- **Retention:** Minimal. One retained pointer: "run `kb/instructions/run-review-batches.md` with the `frontmatter` bundle" — correct in the source repo, broken in installed KBs, where the shipped file lives at `kb/commonplace/instructions/run-review-batches.md` (scaffold trees, `src/commonplace/scaffold_manifest.py:56`; the user's `kb/instructions/` is scaffolded empty). The domain-boundary handoff — the skill's best oracle-warrant feature — dangles in exactly the installed context the promotion mechanism exists to serve.
- **Findings:**
  1. **Installed-KB path break** in the judgment-check handoff (above). Grounded in [retrieval-failure-is-reflection-failure](../../../notes/retrieval-failure-is-reflection-failure.md): the artifact that routes out-of-domain work is unfindable from the runtime copy's own instruction.
  2. **Conditional (parked `warn` placement).** Incumbent-sweep verdicts, audited under both readings. Under **A** (gates are search over incumbents): validator errors/warnings on committed notes are mechanized problem-noticing feeding the fix queue; automating them is the safe automate-first move, and the current design is correct as-is — no change. Under **B** (retain-with-obligation): a standing `ERROR` on an operative note is a retained debt with no tracking consumer — nothing distinguishes a note that has failed validation for a month from one never swept — and B would suggest giving standing failures a durable consumer (a failing-note listing surfaced at write/connect time, or store-side bookkeeping). The verdicts differ, so this stays conditional per the brief.
- **Disposition:**
  1. `apply` — change the final line to: "For judgment-based frontmatter checks, run the `run-review-batches` instruction with the `frontmatter` bundle (`kb/instructions/run-review-batches.md` in this repo; `kb/commonplace/instructions/run-review-batches.md` in installed KBs)."
  2. `conditional` — resolves with the empirical check in [warn-outcome-vs-loop-evaluation](../warn-outcome-vs-loop-evaluation.md); no action until then.

## kb/instructions/cp-skill-convert/SKILL.md

- **Authority:** Agent via harness skill loading, instruction force, `model: sonnet`.
- **Loop function:** **Retention-strengthening**, and nothing else: it converts a raw text capture into a typed, queryable note — adding the retrieval wires (description-as-retrieval-filter, path-valued `type:`), aligning filename to title, and repairing backlinks (wire repair in the [retrieval-failure](../../../notes/retrieval-failure-is-reflection-failure.md) sense). It implements no search and explicitly defers all evaluation ("trait assignment is semantic work, done later"; "Do NOT modify the body content").
- **Oracle warrant:** The refusal to grant `user-verified` ("conversion structures the note but cannot grant human attestation") is exactly right — a mechanical transform claims no more than it establishes. The problem is the reverse direction: the skill *attributes* semantic judgment to oracles that do not perform it — see finding 1.
- **Retention:** The deferred obligations (traits, tags, title quality) are retained only as text in the skill's own report block and in handoff sentences naming consumers that, by their own contracts, will not do the work: line 55 "trait assignment is semantic work, done later by the `cp-skill-validate` skill or human review" — but `cp-skill-validate` is deterministic and instructed "Do not interpret, supplement, or override"; line 56 "`tags` is always `[]` — tag assignment is semantic work, done later by the `cp-skill-connect` skill or human review" — but connect **never mutates artifacts** ("Never edit the target artifact…"); line 63 "whether the title itself is good is a semantic question for the `cp-skill-validate` skill" — same misdirection; and the Step 4 next-steps block repeats it ("Run the `cp-skill-validate` skill … — check quality"). Per [behavioral authority](../../../notes/definitions/behavioral-authority.md), declared intent is not consumption: the named computational consumers have no channel for this work, so the obligation's only real carrier is "human review", and the sentences actively misroute an agent that follows them.
- **Findings:**
  1. **Misattributed handoffs** (four spots above). The theory ground is threefold: a retained obligation with no real consumer leaves the [proposal-selection loop](../../../notes/a-proposal-selection-loop-requires-search-evaluation-and-retention.md) open ("an approved patch that is never merged has no channel"); attributing semantic judgment to the deterministic validator claims more than [its oracle establishes](../../../notes/warranted-autonomy-is-bounded-by-oracle-domain.md); and an agent obeying the pointer performs a retrieval action that cannot succeed — [retrieval failure is reflection failure](../../../notes/retrieval-failure-is-reflection-failure.md).
- **Disposition:**
  1. `apply` — exact edits: line 55 → "`traits` is always `[]` — trait assignment is semantic work, done later by semantic review (the review-batch pipeline) or human review."; line 56 → "`tags` is always `[]` — tag assignment is semantic work, done later by a write/edit informed by a `cp-skill-connect` report, or by human review."; line 63 → "The filename should match the title — whether the title itself is good is a semantic question for later review, not for this conversion."; Step 4 next-steps → "Run the `cp-skill-validate` skill on `new-filename.md` — check structure" and keep the existing human-verification line.

## kb/instructions/cp-skill-ingest/SKILL.md

- **Authority:** Agent via harness skill loading, instruction force, `model: opus`. Composes two other skills by delegated invocation (`cp-skill-snapshot-web`, `cp-skill-connect`) — channel composition the type contract (`kb/instructions/COLLECTION.md`, `invokes` label) authorizes.
- **Loop function:** **Search** for the KB-content loop: it brings an external source under consideration and distills goal-relative candidate value ("extract goal-relative value… recommend one advisory next action"). The relevance judgment ("If the source is not relevant to this KB, say so… recommend no promotion") is search-side pruning under the [strict criterion](../../../notes/a-proposal-selection-loop-requires-search-evaluation-and-retention.md): the snapshot stays retained and the report is written either way; the verdict shapes only what candidates the report recommends. The `.ingest.md` itself is retained — but as a knowledge artifact (evidence/advice force), not a binding commitment.
- **Oracle warrant:** Respected by keeping force at advice: "Recommend exactly one **advisory** next action" — the model's judgment never functions as an acceptance gate, so no oracle warrant is claimed beyond structural validation (Step 6, scoped correctly). Heavy model automation, entirely on the reject-safe side of [the asymmetry](../../../notes/false-positive-generation-is-filtered-before-retention.md).
- **Retention:** Strong discipline in two places. (1) The regenerable connect report must not leak into the durable artifact: "Do not cite it, link to it, or name its path in the durable ingest report" — durable artifacts are not wired to regenerable intermediates, matching the derived-artifact maintenance reasoning in the [two-layer note](../../../notes/theory-and-methodology-form-a-two-layer-execution-system.md). (2) The no-casebook rule refuses to pre-write relationships "that go stale the moment notes are written" — staleness-aware refusal to retain, with the recompute correctly assigned to a future connect run.
- **Findings:** No change suggested. Second positive exhibit for the digest: automate-search-first with advisory output and explicit staleness refusals.
- **Disposition:** n/a (no findings).

## kb/instructions/cp-skill-snapshot-web/SKILL.md (+ KNOWN-LIMITATIONS.md)

- **Authority:** `SKILL.md`: agent via harness skill loading, instruction force, `model: sonnet`; also invoked by `cp-skill-ingest`. `KNOWN-LIMITATIONS.md`: copied with the skill by the promotion mechanism, but **referenced by nothing** — a repo-wide grep for `KNOWN-LIMITATIONS` finds only the file itself. Its actual behavioral authority is none: no consumer, no channel, no force.
- **Loop function:** **Search-side acquisition** (capture into `kb/sources/`) plus one genuine small **evaluation** gate: the Step 1 duplicate check, whose verdict controls a block ("tell the user and stop") — an operation distinct from producing the next candidate, so it passes the strict criterion. **Retention**: the snapshot itself, immutable by convention, with retrieval wires in frontmatter (source URL, description-as-retrieval-filter, `captured`, `capture`, `genre`).
- **Oracle warrant:** Two oracles. The duplicate gate's oracle is an exact-string URL grep; its domain excludes URL variants (http/https, trailing slash, mobile/AMP mirrors, arxiv `abs` vs `pdf` of the same paper), so "not found" establishes less than "not already captured" — acceptance mildly exceeds the check, but the failure mode is a benign visible duplicate. The `genre` field is scoped correctly ("a value outside it validates with a warning… a surface judgment… ingestion may correct it later").
- **Retention:** The skill's most consequential retained commitment lives in `KNOWN-LIMITATIONS.md`: WebFetch summarizes long pages; `capture: web-fetch` marks affected snapshots; the workaround is the PDF URL for arxiv; the detection heuristic is "<~100 lines of a long-form source". None of it is loaded when it matters: Step 2 routes `arxiv.org/html/` pages to WebFetch — **the exact case the file warns about** — and nothing at capture or ingest time surfaces the completeness heuristic. This is the [retrieval-failure note](../../../notes/retrieval-failure-is-reflection-failure.md)'s definition of an inert represented constraint: "it is written, it is true, and it changes nothing." The `capture:` frontmatter mark is a fine wire on the snapshot side, but the knowledge of what the mark *means* has no consumer.
- **Findings:**
  1. **KNOWN-LIMITATIONS.md has no retrieval wire.** Grounded above. Fix direction follows the collection's own frontloading posture (`kb/instructions/COLLECTION.md`: "Self-contained… don't assume the reader has loaded other KB documents"): frontload the two operative rules into `SKILL.md` rather than adding a read instruction.
  2. Duplicate-oracle domain narrowness (above). Real but low-stakes.
- **Disposition:**
  1. `apply` — two edits to `SKILL.md`: (a) extend the Step 2 PDF trigger from "URL ends in `.pdf`, or `arxiv.org/pdf/`" to also match `arxiv.org/abs/` and `arxiv.org/html/`, rewriting those to the corresponding `arxiv.org/pdf/` URL before Step 2c (rationale line: WebFetch summarizes long documents; the PDF pathway reads in full); (b) in Step 2d / Step 5, add: "If a web-fetch capture of a long-form source comes back under ~100 lines, it was likely summarized — tell the user and suggest the PDF URL or manual paste." Keep `KNOWN-LIMITATIONS.md` as the explanation and reference it from the new rationale line, or fold it in and delete it (either restores the wire).
  2. `reject` — URL normalization for the duplicate grep is not warranted by observed traffic; the failure is a visible, cheap-to-reconcile duplicate (filtered downstream by humans), and the normalization surface (mirror hosts, tracking params) is open-ended. Revisit only if duplicate snapshots recur.

Flag the two non-ADR out-of-group occurrences for the same sweep.

## Group summary

Findings most worth the maintainer's attention:

1. **A class, not just instances: promoted skills retaining paths that break on their real authority path.** Three separate wires fail from the runtime copies: `cp-skill-write`'s markdown-relative `../../reference/link-vocabulary.md`, `cp-skill-validate`'s `kb/instructions/run-review-batches.md` (installed KBs only), and `cp-skill-health-check`'s `PROMOTED_SKILLS`-in-`init_project.py` pointer. The rule that would prevent all three exists — but as advice in `kb/instructions/README.md`, which nothing loads at skill-edit time. Per [false-positive-generation-is-filtered](../../../notes/false-positive-generation-is-filtered-before-retention.md), this is a cheap deterministic oracle waiting to be built: **proposal** — a validator check on promoted-skill `SKILL.md` files (no markdown-relative links; flag repo-root paths that differ between source-repo and installed layouts; optionally verify that auxiliary files in a promoted skill directory are referenced from `SKILL.md`). That last clause would also have caught finding 2.
2. **`cp-skill-snapshot-web`'s KNOWN-LIMITATIONS.md is inert** — retained knowledge with no consumer, while the SKILL.md routes arxiv HTML pages into the exact failure the file documents. Disposition `apply` (frontload the two rules).
3. **`cp-skill-convert` misroutes its deferred obligations** to computational consumers (`cp-skill-validate`, `cp-skill-connect`) whose own contracts forbid the attributed work. Disposition `apply` (four sentence edits).
4. **Positive result, equally load-bearing for the digest:** `cp-skill-connect`, `cp-skill-ingest`, and `cp-skill-health-check` already implement the cluster's automate-search-first allocation cleanly — model automation concentrated in search, advisory output, human at the accept gate, and (health-check, validate) explicit oracle-scoping language. The digest can cite these as worked instances rather than prescribing from scratch.
5. **Authority path 1 has a natural attachment point that costs no new mechanism.** `cp-skill-write` hard-fails without reading the target collection's `COLLECTION.md`; when the artifact being edited is an instruction, gate, or contract, that file is loaded at exactly change time. A change-time digest reachable from (or frontloaded into) `kb/instructions/COLLECTION.md` — and the reference/types contracts likewise — would ride an existing enumeration-grade wire instead of the best-effort discovery path the workshop README worries about. Currently **nothing** in any of the seven skills routes system-definition edits to the cluster.

Observations for the phase-1 scheduled checks:

- **(a) Cumulativity's environment-mediated exclusion:** no case in this group pressures the stated form. Where these skills are non-cumulative it is deliberate and representation-layer-shifted, not environment-mediated: connect reports are regenerated from current KB state each run, with cumulativity carried by the library artifacts the runs read and write; snapshot-web's duplicate gate consumes earlier retention directly. Improvement flows through retained representations throughout — the included kind.
- **(b) Reflection vs self-improvement routing:** these are operational artifacts, not the cluster's routing artifacts, and none carries cluster traffic; the only tag-head consumption in the group is `cp-skill-connect` reading tag-READMEs (including enforced `complete:` marks) for prospecting — routing consumption that is real but topic-agnostic. Nothing here bears on the axis-factoring decision beyond confirming that skill-side traffic reaches tag heads through connect's prospecting pass, so a minted `reflective-systems` head would be consumed by the same mechanism at no extra wiring cost.
- **(c) "Promotion" vocabulary:** four uses encountered. Skill promotion (`kb/instructions/README.md`, health-check, the manifest): instruction directory → runtime skill surfaces — this sense *creates* the consumer and channel, i.e., it is authority-promotion in the constraining sense, the most binding of the family. Connect's Maintenance Observations: "promote entries… into durable local artifacts" — durability promotion. Ingest: "recommend no promotion or source-only filing" — source → library, also durability. Convert's future "any → text: strip frontmatter, **demote** to raw capture" — the family's inverse, confirming the shared axis. All four fit "move into a more binding or durable layer"; no collision observed, but the skill-promotion sense is distinct enough (channel-creating, not merely durability-increasing) that a write-time registration line distinguishing it would be cheap insurance under the [collision rule](../../../notes/vocabulary-collisions-prevented-at-write-time-not-read-time.md).

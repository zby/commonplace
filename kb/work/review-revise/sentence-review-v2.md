# Sentence-Level Review v2 (experiment)

**Target: kb/work/review-revise/baseline.md**

## What this is

A read-only review that reads sentence by sentence, looking for local defects. The existing prose review checks for note-level representational patterns. This review checks whether individual sentences say what they mean.

Report findings as WARN (likely misleads the reader) or INFO (mildly unclear), never FAIL.

## Prerequisites

Read the target note in full. Note: this note originally lived at `kb/notes/session-history-should-not-be-the-default-next-context.md`. Resolve links relative to `kb/notes/` when checking link targets.

## Checks

### 1. Parsing ambiguity

**Failure mode:** A sentence can be read two ways due to its syntactic structure, and the unintended reading is plausible.

**Test:** For each sentence, ask: is there a way to read this that means something different from what was intended? Pay special attention to:
- Negation scope: "The mistake is not storing a trace" — does "not" negate "mistake" or "storing"?
- Modifier attachment: which verb does a "when" or "where" clause attach to?
- Pronoun reference: is the antecedent clear?

Only report cases where the unintended reading is plausible.

### 2. Framing mismatch

**Failure mode:** A sentence frames its claim in terms of X, but the actual mechanism is Y. The framing narrows or misdirects the reader's understanding.

**Test:** For each sentence that gives a reason or explains a mechanism, ask: is the stated framing the right level of generality? Is the mechanism correctly identified?

**Example:** "For orchestration, that is usually the wrong trade" — frames as orchestration-specific, but the mechanism (context degradation) applies to any LLM call.

### 3. Stock phrases

**Failure mode:** A sentence uses a pattern commonly generated by LLMs that adds no content.

**Primary test: delete the sentence.** If the paragraph reads equally well or better without it, the sentence is filler regardless of its pattern. This deletion test is the main criterion — pattern matching is secondary.

**Common patterns that warrant the deletion test:**
- "This is not just X — it is Y" (the "elevation" move)
- "It's worth noting that..." / "It's important to note that..."
- "In other words, ..." followed by a near-verbatim restatement
- "At its core, ..."
- "The key insight here is that..."

**Severity:** If the deletion test shows the sentence is removable, report as WARN, not INFO. The previous version was too lenient on borderline cases — if surrounding text already establishes the point, the stock-phrase sentence is filler even if it "adds something."

### 4. Misleading link text

**Failure mode:** The text of a markdown link implies a concept, but the linked note discusses something different.

**Test:** For each markdown link, read the link text and the sentence it appears in. What does the reader expect to find? Then read the target note's title and opening paragraph (resolve relative to `kb/notes/`). Does the target match?

## Output

Write the review to the run directory using this format:

```
=== SENTENCE-LEVEL REVIEW v2: baseline.md ===

Checks applied: 4

WARN:
- [{check-name}] {finding with specific quote from the note}
  Recommendation: {what to change}

INFO:
- [{check-name}] {finding}

CLEAN:
- [{check-name}] {what was checked and why it held}

Overall: {CLEAN / {N} warnings, {M} info}
===
```

Report every check — WARN, INFO, or CLEAN.

## Do NOT

- Do not modify the target note. This is read-only analysis.
- Do not report note-level patterns (proportion, structure, completeness). Those belong to other reviews.
- Do not follow more than 5 links for the misleading-link-text check.

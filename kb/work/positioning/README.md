# Workshop: Positioning

## Question

How do we frame Commonplace as a business tool for early adopters who are already using AI agents seriously?

## Context

Commonplace is a framework for building agent-operated knowledge bases. The methodology is mature enough to be useful, but the positioning hasn't been articulated for an external audience. The challenge: "knowledge base" sounds like a commodity (Notion, Confluence, wikis). We need a frame that communicates the actual value — structured knowledge that makes AI agents reliably better over time.

Target audience: teams and individuals already committed to AI agents, who have hit the wall where agents hallucinate, forget, or produce shallow work due to lack of context.

## What we want to discover

- What framing resonates? "The knowledge layer for AI agents" vs "context engineering for teams" vs something else
- What's the minimum viable pitch? What can we cut while keeping the core insight
- What artifacts does this workshop produce? Landing page, README pitch, talk outline, one-pager?
- How do we demonstrate value without requiring deep methodology buy-in?

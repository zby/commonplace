# Investigation: two-axis agent-memory taxonomy (content-type × hierarchy-level)

## Claim under test

From `kb/log.md` (SYNTHESIS entry, also carried in the current triage README as "the strongest promotion candidate"):

> Two independent decompositions of agent memory from different traditions (cognitive science vs computer architecture) that together predict a two-axis taxonomy: content type × hierarchy level.

The two decompositions:

1. **Content-type axis** (cognitive science / Tulving) — three-space split of agent memory into knowledge / self / operational, echoing Tulving's semantic/episodic/procedural taxonomy.
2. **Hierarchy-level axis** (computer architecture) — three-layer split of I/O / cache / memory, by access latency, capacity, and persistence.

The proposed synthesis: these are two *independent* dimensions, so a genuine product-space taxonomy exists (which content types belong at which hierarchy levels), and this joint model would predict architectural decisions that neither axis alone can.

## What I read

- `kb/notes/three-space-agent-memory-echoes-tulvings-taxonomy-but-the-analogy.md` (full note, status: `speculative`). Its own table maps Tulving's semantic/episodic/procedural to knowledge-graph/self-space/operational-space, and — critically — a third column labels each row's **"metabolic rate"**: steady growth / slow evolution / high churn. The note's body says explicitly: "The key insight is not just that these are different *topics* but that they have different *lifecycles*," and closes by conceding "the mapping to Tulving is suggestive but may be decorative. The practical value could reduce to simpler advice: separate persistent knowledge from transient working files, and give them different retention policies." The note already treats content-type as possibly reducible to a lifecycle/rate variable, not as an independent axis.
- `kb/sources/multi-agent-memory-computer-architecture-perspective.md` and its `.ingest.md` (full text, both). The paper is a non-empirical position paper about **multi-agent** memory *consistency* (shared vs. distributed paradigms, cache-coherence-style protocols across agents). Its three-layer hierarchy (I/O / cache / memory) is defined by access characteristics: I/O is the ingestion/emission interface, cache is fast/limited-capacity working memory, memory is large/persistent storage. The ingest's own Limitations section is candid that: "No mapping validation" exists (no system is analyzed in enough depth to show the hierarchy is productive rather than superficial); the shared/distributed split is "less crisp than claimed"; and, most relevant here, "No engagement with the cognitive science tradition" — the paper never touches content type at all, so it offers no argument that hierarchy-level and content-type are distinct, independently-varying dimensions.
- The ingest's "Connections Found" section is where the two-axis idea originates: it explicitly frames this as a "Synthesis opportunity" and an "[experiment]" — "worth testing whether the two-axis model... produces better architectural predictions than either axis alone." It was never claimed as established by either source; it's a hypothesis our own connect tooling generated by juxtaposing the two notes.

## Search for existing coverage

Checked `kb/notes/agent-memory-README.md`, `kb/agent-memory-systems/README.md`, and grepped `kb/notes/`, `kb/agent-memory-systems/`, `kb/reference/` for "two-axis" / "hierarchy level" / "content type." No existing note claims this joint structure for agent memory. Three incidental hits are unrelated: `short-composable-notes-maximize-combinatorial-discovery.md` merely *cites this same log entry* as a worked example of cross-domain discovery (not as an endorsed claim — it doesn't evaluate whether the taxonomy holds); `access-burden-and-transformation-burden-are-independent-query.md` and `charting-the-knowledge-access-problem-beyond-rag.md` use "two-axis" for an unrelated query-dimension claim (transformation burden × access burden), not memory taxonomy; `xMemory.md` uses "hierarchy levels" as an implementation detail of one reviewed system, not a taxonomy claim. No duplication risk either way.

## Reach test (`kb/notes/COLLECTION.md`)

- **Change one premise, predict the conclusion change?** Cannot do this reliably. Within the Tulving note's own table, content-type and lifecycle-rate move in lockstep (semantic↔steady-growth, episodic↔slow-evolution, procedural↔high-churn) — there is no case in either source where a single content type is shown at more than one hierarchy level, or a single hierarchy level hosting multiple content types independently of rate. Without that cross-variation, there's no basis to predict what changes when one axis is held fixed and the other varies — because it hasn't been shown the axes vary independently at all.
- **Would it apply in a different domain?** The generic version — "separate what you store (content) from how fast/how persistently you store it (tier)" — is standard caching/tiered-storage doctrine, not a novel insight specific to agent memory. The specific claim (this content-type triad crosses this hierarchy-level triad in agent systems) has no evidence outside these two sources.
- **Could someone say exactly how it's wrong, not just incomplete?** Yes, concretely: the two "axes" may be two vocabularies for the *same* latent variable (turnover rate / access frequency), not orthogonal dimensions — procedural/operational content is high-churn *and* cache-like; semantic/knowledge content is steady-growth *and* memory-like. If that's right, the "two-axis taxonomy" over-counts a single dimension as two, which is a specific, falsifiable way the synthesis could be wrong (as opposed to merely under-elaborated). Conversely, the CS paper's I/O layer (an interface/boundary concept) doesn't correspond to any lifecycle rate at all, so the mapping is at best a loose 2-of-3 correspondence, not a clean product space either. Either way — collapse to one axis, or a mapping too ragged to call a taxonomy — the joint claim isn't well-supported by what's actually in the two sources.

## Verdict: DISMISS

The apparent convergence is a resemblance manufactured by our own connect tooling juxtaposing two independently-hedged, non-empirical sources — not a claim either source argues for, and not one that survives inspection once the sources are read in full. The Tulving note already flags its own content-type axis as possibly decorative/reducible to a lifecycle variable; the CS paper never engages content type at all and is explicit that its own hierarchy mapping is unvalidated. Combining two unvalidated, possibly-collapsing decompositions does not produce a validated joint one — it compounds the uncertainty. Writing the suggested note ("Agent memory needs both a content-type axis and a hierarchy axis") would present as a KB claim something no source demonstrates and that a straightforward reading of the existing material argues against (same-axis relabeling).

This is a case where "a synthesis flagged in the log is a candidate, not a foregone conclusion" — and where the well-argued answer is not to write the note, despite two prior triage passes flagging it as high-potential.

**Reopening condition**, for anyone revisiting this later: the claim would become writable if a concrete case surfaces where the same content type is hosted at more than one hierarchy level (or vice versa) in an actual reviewed system — i.e., evidence that the two variables genuinely vary independently rather than moving together. Absent that, this stays closed. (Note, not adopted here as new work: the sharper and better-supported claim latent in this investigation — that two independently-sourced decompositions can look like orthogonal axes while actually re-describing one latent variable — would itself pass the reach test and could be a note on its own, grounded in this case as a worked example. Flagging it for a future triage decision rather than writing it now, since it's outside this item's assigned scope.)

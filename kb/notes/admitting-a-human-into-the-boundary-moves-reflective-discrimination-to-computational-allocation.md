# Admitting a human into the boundary moves reflective discrimination to computational allocation

A [reflective system](./definitions/reflective-system.md) makes its self-representation available to processes inside a declared boundary; those processes need not be computational. Include a maintainer with a standing causal role, and nearly any maintained system with readable source can qualify: the maintainer inspects the source as a representation, edits it, and builds carry the edit into later behavior. The attribution is not false, but it is cheap.

What remains discriminating is who performs each improvement function. Under a fixed human-inclusive boundary, report whether consequential decisions are human, computational, or joint. The composite may be reflectively [self-improving](./definitions/self-improving-system.md) while only some functions run computationally.

In the tag-README pathway, a maintainer chose the problem and accepted the type split, the maintainer and agent jointly framed the candidate, and a validator recomputed exact tag membership. The same human-inclusive system therefore has human, joint, and computational functions in one improvement episode.

If the initial task description counts as participation in every downstream decision, almost no agent function is computational; if it never counts, the profile hides how much decision content the instruction supplied.

The same allocation has a closure reading: a function is computationally closed when it needs no human decision; whole-pathway computational closure is the endpoint at which every required function meets that condition. This is machine autonomy viewed from the dependency side, not a separate grade. It is also distinct from methodological closure: a human can execute a settled method, while an unattended model can improvise, [because the two closure readings track different changes](./methodological-and-computational-closure-track-different-changes.md).

A profile is more honest than a percentage because search, evaluation, adoption, retention, and revision differ in both decomposition and stakes. Where two systems admit a common decomposition, one is more computationally autonomous on a pathway when it assigns computational components at least the same functions with at least the same authority and warrant, plus some additional function. Otherwise the profiles may be incomparable.

Warrant stays separate: replacing a human evaluator with an unconstrained model changes the allocation without establishing that its acceptances can be trusted, [since warranted autonomy is bounded by oracle domain](./warranted-autonomy-is-bounded-by-oracle-domain.md).

For Commonplace, designated maintainers make the socio-technical system reflectively self-improving; [the case study](../reference/commonplace-as-a-reflective-system.md) therefore reports the more informative split among maintainers, agents, and validators.

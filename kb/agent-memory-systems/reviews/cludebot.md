# Clude

Clude, by sebbsssss, is a TypeScript monorepo for an AI memory product: a Cortex SDK, MCP server, hosted API, chat application, dashboards, memory-pack tooling, and an agent-oriented memory loop. The advertised core is "cognitive memory" rather than passive storage: typed memories decay, retrieval combines semantic and symbolic signals, and "dream" phases derive semantic, procedural, self-model, contradiction-resolution, and emergence memories from stored episodes ([README.md](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/README.md), [packages/brain/src/memory/memory.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/memory.ts), [packages/brain/src/memory/dream/cycle.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/dream/cycle.ts)).

## Core Ideas

**The central retained object is a typed memory row, not a note file.** Clude stores memories as typed records with content, summary, tags, concepts, importance, emotional valence, decay, source metadata, owner scope, evidence IDs, hash IDs, optional encryption, and optional on-chain commitment columns. The main Supabase schema defines the hosted/self-hosted substrate, while the MCP SQLite mode creates an analogous local `memories`, `links`, and `dream_queue` schema under `~/.clude/brain.db` ([packages/database/supabase-schema.sql](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/database/supabase-schema.sql), [packages/brain/src/storage/schema.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/storage/schema.ts), [packages/brain/src/storage/sqlite-store.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/storage/sqlite-store.ts)).

**Retrieval is a multi-signal ranking pipeline.** `recallMemories` expands queries when configured, runs vector search over memory rows and fragments, mixes in metadata candidates, BM25, knowledge seeds, entity graph matches, co-occurring entities, bond-typed graph traversal, type diversity, owner filtering, access tracking, and co-retrieval reinforcement. Its score combines recency, keyword/tag relevance, importance, vector similarity, BM25, memory type boosts, knowledge-seed boosts, and penalties for internal generated sources ([packages/brain/src/memory/memory.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/memory.ts), [packages/database/supabase-schema.sql](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/database/supabase-schema.sql), [packages/brain/src/experimental/enhanced-recall.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/experimental/enhanced-recall.ts)).

**Context efficiency is handled by summaries, fragments, and token-budgeted read-back.** Memories have short summaries, full content hydration, and per-fragment embeddings for long content. The SDK exposes `recallSummaries` and `hydrate` as progressive disclosure. Hosted chat recalls up to 25 candidates from the user message, merges temporal hits, then `fitToTokenBudget` keeps the system prompt, recent history, and only as many memories as fit; memory context is grouped into knowledge, behaviors, identity, and recent sections ([packages/brain/src/sdk/cortex.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/sdk/cortex.ts), [packages/brain/src/memory/memory.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/memory.ts), [apps/server/src/routes/chat.routes.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/apps/server/src/routes/chat.routes.ts)).

**Dream cycles are trace distillation, not just maintenance.** When episodic memories are stored, an event bus can accumulate importance and trigger a dream cycle. The cycle generates focal questions from recent memories, retrieves evidence, stores semantic consolidation memories with evidence IDs, extracts procedural insights, compacts faded episodes, reflects into self-model memories, resolves contradictions, runs action-outcome learning, and stores dream logs ([packages/brain/src/sdk/cortex.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/sdk/cortex.ts), [packages/brain/src/memory/dream/cycle.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/dream/cycle.ts), [packages/brain/src/memory/action-learning.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/action-learning.ts)).

**MCP and SDK integration make memory easy to adopt, but authority varies by host.** The MCP server exposes `recall_memories`, `store_memory`, `get_memory_stats`, `find_clinamen`, delete/update/list/batch operations, and skill extraction across hosted, self-hosted, SQLite, and JSON local modes. The setup command installs MCP entries and can inject a `CLAUDE.md`/`AGENTS.md` section telling agents to recall at session start and store useful memories ([packages/brain/src/mcp/server.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/mcp/server.ts), [packages/brain/src/mcp/local-store.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/mcp/local-store.ts), [packages/brain/src/cli/setup.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/cli/setup.ts), [SKILL.md](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/SKILL.md)).

**The product also dogfoods its own memory as behavior policy.** The repository-level `AGENTS.md` requires every agent to recall and store memories, and the checked-in `clude-memories.json` contains operational memories from action logging, outcome tracking, consolidation, and self-reflection. That file is not a source-of-truth database for hosted Clude, but it is relevant evidence of the memory schema being used for action/outcome/self-model traces ([AGENTS.md](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/AGENTS.md), [clude-memories.json](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/clude-memories.json)).

## Artifact analysis

- **Storage substrate:** `sqlite` — Supabase `memories` rows in hosted/self-hosted mode, SQLite `memories` rows in the default local MCP mode, and JSON entries in explicit local JSON mode
- **Representational form:** `mixed` — Mixed. The content and summary are prose; type, tags, concepts, decay, importance, access counters, owner, source, evidence IDs, encryption state, hash IDs, and commitment fields are symbolic

**Memory records.** Storage substrate: Supabase `memories` rows in hosted/self-hosted mode, SQLite `memories` rows in the default local MCP mode, and JSON entries in explicit local JSON mode. Representational form: mixed. The content and summary are prose; type, tags, concepts, decay, importance, access counters, owner, source, evidence IDs, encryption state, hash IDs, and commitment fields are symbolic. Lineage: authored by MCP/SDK/API calls, imported from memory packs and chat exports, generated by chat turns, or derived by dream, reflection, action-learning, and compaction paths. Behavioral authority: knowledge artifact when recalled as evidence or context; system-definition artifact when procedural and self-model memories are inserted into prompt sections that say they should guide behavior ([packages/brain/src/memory/memory.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/memory.ts), [apps/server/src/routes/chat.routes.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/apps/server/src/routes/chat.routes.ts), [packages/brain/src/mcp/local-store.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/mcp/local-store.ts)).

**Embeddings, fragments, BM25 tokens, and ranking state.** Storage substrate: Supabase memory embeddings, `memory_fragments`, lexical token tables, HNSW/RPC functions, in-process caches, and SQLite `memory_embeddings` virtual tables when `sqlite-vec` is available. Representational form: mixed symbolic plus distributed-parametric. Lineage: generated from memory summaries, content chunks, tag/context fragments, and query embeddings; regenerated by re-embedding scripts or write-time embedding. Behavioral authority: system-definition artifact with ranking and selection authority, because it decides which memories enter context before a model acts ([packages/brain/src/memory/memory.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/memory.ts), [packages/database/supabase-schema.sql](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/database/supabase-schema.sql), [packages/brain/src/storage/sqlite-store.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/storage/sqlite-store.ts)).

**Memory graph, entity graph, and trace graph.** Storage substrate: `memory_links`, entities, entity mentions, entity relations, and derived trace walks over evidence IDs and links. Representational form: symbolic graph plus optional entity embeddings. Lineage: memory links are authored or generated by auto-linking, dream phases, co-retrieval reinforcement, action/outcome linking, contradiction resolution, and JEPA deep-connection phases; entities are extracted from memory content. Behavioral authority: ranking and explanation authority. Graph paths can boost recall and can explain why a memory exists, but they do not by themselves enforce behavior ([packages/brain/src/memory/memory.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/memory.ts), [packages/brain/src/memory/graph.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/graph.ts), [packages/brain/src/memory/trace.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/trace.ts), [packages/database/migrations/003_entities.sql](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/database/migrations/003_entities.sql)).

**Dream logs and derived memories.** Storage substrate: Supabase `dream_logs` plus the new semantic, procedural, self-model, and contradiction-resolution memory rows produced by dream phases. Representational form: prose outputs with symbolic session type, input memory IDs, output memory IDs, evidence IDs, and link records. Lineage: trace-derived from stored episodic and semantic memories, action/outcome memories, self-model memories, contradiction pairs, and retrieval results; the LLM output parser tries to map citations back to source memory IDs. Behavioral authority: dream logs are knowledge artifacts for audit; derived procedural/self-model/semantic memories gain advisory or instruction-like authority when later recalled and formatted into prompt context ([packages/brain/src/memory/dream/cycle.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/dream/cycle.ts), [packages/brain/src/memory/action-learning.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/action-learning.ts), [packages/database/supabase-schema.sql](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/database/supabase-schema.sql)).

**MCP tools, SDK methods, hosted routes, and chat prompt assembly.** Storage substrate: package code, installed MCP config, hosted Express routes, and generated/injected instructions. Representational form: symbolic tool schemas and route logic plus prose instructions and formatted memory sections. Lineage: authored integration code and operator setup. Behavioral authority: system-definition artifact. MCP tools give agents pull access; hosted chat and `buildAndGenerate` give runtime push authority by retrieving memories before generation and placing them in system/prompt context ([packages/brain/src/mcp/server.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/mcp/server.ts), [packages/brain/src/sdk/cortex.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/sdk/cortex.ts), [packages/brain/src/services/response.service.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/services/response.service.ts), [apps/server/src/routes/chat.routes.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/apps/server/src/routes/chat.routes.ts)).

**Memory packs, exports, and prompt sync files.** Storage substrate: exported JSON/Markdown/ChatGPT/Gemini/MemoryPack files, SDK `MemoryPack` objects, and optional generated prompt files. Representational form: mostly prose plus symbolic manifests/records. Lineage: exported from selected or recalled memories, privacy-filtered on export, imported with source and tag modifications, or synthesized into prompt context. Behavioral authority: knowledge artifact when transferred as portable context; system-definition artifact when a synced prompt file is wired into another agent's always-loaded system prompt ([packages/brain/src/sdk/cortex-v2.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/sdk/cortex-v2.ts), [packages/brain/src/cli/export.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/cli/export.ts), [packages/brain/src/cli/import.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/cli/import.ts), [packages/brain/src/cli/sync.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/cli/sync.ts)).

The promotion path is explicit but distributed: a chat/action/social trace can become an episodic memory; dream and action-learning phases can distill it into semantic, procedural, or self-model memories; retrieval indexes and graphs can increase its read-back probability; hosted chat can promote a recalled procedural memory into a prompt section headed "behaviors." The weak point is review governance. Importance, decay, links, evidence IDs, and confidence gates help rank and explain memory, but the code does not define a separate human-reviewed state for "this derived memory is now trusted policy."

## Comparison with Our System

| Dimension | Clude | Commonplace |
|---|---|---|
| Main substrate | Database rows, embeddings, graphs, local SQLite/JSON, exported packs | Git-tracked Markdown collections, type specs, validation, reviews |
| Primary reader | Running agents and hosted chat sessions | Agents and maintainers navigating a knowledge base |
| Memory creation | SDK/MCP/API writes plus chat/action/dream/reflection loops | Authored notes, reviews, indexes, snapshots, skills, validation outputs |
| Context efficiency | Summary/full hydration, fragment embeddings, top-k recall, token-budget fitting | Lexical search, curated/generated indexes, authored links, task-scoped loading |
| Trace learning | Strong: action/outcome/chat traces become durable derived memories | Present but more explicit and review-mediated |
| Read-back | Pull tools plus automatic chat/API prompt injection | Mostly pull/navigation unless a skill or instruction explicitly loads context |
| Governance | Owner scoping, encryption/revocation, source fields, evidence IDs, confidence gates | Frontmatter status, type contracts, schema validation, semantic review gates |

The strongest alignment is that both systems treat retained artifacts as operational infrastructure, not just user notes. Clude makes this obvious at runtime: memory rows, links, embeddings, dream logs, and injected prompt sections directly shape responses. Commonplace makes it explicit in review and maintenance: collection contracts, types, validation, and source-pinned reviews define how artifacts should be trusted.

The largest divergence is substrate and authority discipline. Clude is optimized for live agents, so it accepts a broad mix of database records, generated insights, embeddings, graph edges, and prompt assembly. Commonplace is slower and more legible: durable behavior-shaping artifacts are mostly Markdown or code, with visible lineage and review status.

Clude's context-management story is more automated. The hosted chat path starts recall before the user message is even fully persisted, merges temporal retrieval, trims history and memories to a token budget, and records which memories were used. Commonplace usually asks the agent to search, scan indexes, and decide what to load. That is more inspectable, but it leaves more activation work to the agent.

**Read-back:** `both` — MCP/SDK recall is pull, while hosted chat and response-service paths perform instance-targeted memory push by retrieving memories from the incoming query/task and injecting selected memories into prompt context before generation

### Borrowable Ideas

**Evidence-linked distillation records.** Ready now as a review pattern. Clude's dream cycle keeps input memory IDs, output IDs, and parsed evidence IDs. Commonplace trace-derived workflows should preserve source trace IDs and derived artifact IDs together, even when the final artifact is prose.

**Prompt budget fitting as a first-class read-back policy.** Ready when Commonplace has runtime context packs. Clude's hosted chat keeps base instructions, recent history, and memory context under an explicit budget. Commonplace could express similar budgets for skill-driven context bundles.

**Separate summary recall and hydration.** Ready conceptually. Clude's summary-first API maps well to Commonplace indexes and descriptions: retrieve cheap selectors first, then hydrate the few artifacts that matter.

**Action/outcome to procedural-memory loop.** Useful but needs governance. Clude can turn repeated action outcomes into procedural memories. Commonplace could use the same path for agent operating lessons, but only if the lesson gets review status before becoming instruction.

**Clinamen as deliberate peripheral attention.** Interesting but not immediately needed. High-importance, low-relevance retrieval is a good creative/review prompt, but Commonplace should use it as a curiosity or review aid, not as default context.

**Do not borrow silent authority escalation.** Clude's derived procedural/self-model memories can become prompt instructions when retrieved. Commonplace should keep the distinction between knowledge artifact, advice, and enforced system-definition artifact explicit.

## Trace-derived learning placement

**Trace source.** Clude qualifies as trace-derived. Its raw signals include user chat turns, assistant replies, action records, outcome records, recent episodic memories, self-model memories, semantic memories, contradiction pairs, social engagement outcomes, and internal reflection outputs. The checked-in sample `clude-memories.json` shows action, outcome, consolidation, self-model, and social-interaction records in the same schema used by the code ([apps/server/src/routes/chat.routes.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/apps/server/src/routes/chat.routes.ts), [packages/brain/src/memory/action-learning.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/action-learning.ts), [packages/brain/src/memory/dream/cycle.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/dream/cycle.ts), [clude-memories.json](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/clude-memories.json)).

**Extraction.** Extraction is multi-stage. Chat routes store full conversation turns as episodic memories. Action logging records the action and reasoning; outcome tracking records later measured results and updates the original action. Dream consolidation generates focal-point questions, retrieves relevant memories, asks an LLM for evidence-cited observations, stores semantic memories, and links them back to sources. Procedural extraction and strategy refinement generate behavioral rules from recent episodic memories or action/outcome rates. Reflection and active reflection write self-model or introspective memories from selected seeds. Contradiction resolution writes semantic resolution memories and accelerates decay on weaker contradicting records ([apps/server/src/routes/chat.routes.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/apps/server/src/routes/chat.routes.ts), [packages/brain/src/memory/action-learning.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/action-learning.ts), [packages/brain/src/memory/dream/cycle.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/dream/cycle.ts), [packages/brain/src/memory/active-reflection.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/active-reflection.ts)).

**Four fields.** The raw stage is mostly episodic memory rows and chat/action/outcome records in database or local store substrate; representational form is prose plus symbolic metadata; lineage is direct trace capture; authority is knowledge artifact evidence. The distilled stage is semantic/procedural/self-model/introspective memory rows, dream logs, memory links, entity links, and prompt sections; representational form is mixed prose/symbolic/embedding; lineage is generated from trace memories through LLM and heuristic phases; authority ranges from advice to prompt instruction and ranking influence. This split is important: the raw trace is not itself the lasting policy. The distilled memory row becomes the behavior-shaping artifact.

**Scope and timing.** Scope is owner-wallet/user/agent scoped in hosted and Supabase modes, local-machine scoped in SQLite/JSON MCP modes, and optionally imported/exported through memory packs. Timing ranges from online write-time storage, fire-and-forget embedding/linking/entity extraction, per-message hosted read-back, event-driven dream triggers, 6-hour fallback dream cycles, daily decay, active reflection schedules, and explicit SDK/CLI calls.

**Survey placement.** Clude sits in the trace-to-operational-memory family: traces become typed memories, distilled insights, behavioral strategies, self-model records, graph edges, and retrieval indexes. It strengthens the survey claim that trace-derived learning usually needs a distillation boundary before it should shape behavior. It also shows the hazard: once distilled, a procedural memory can be injected as a rule-like instruction without a separate review gate.

**Curation policy.** The curation oracle is mixed. LLM calls generate focal questions, insights, reflections, contradiction resolutions, procedural rules, and themes. Heuristics handle deduplication, type filters, thresholds, decay, query expansion fallbacks, source penalties, owner guards, token budgets, and some local retrieval. Human review is available through dashboards, exports, and trace/explain routes, but it is not a mandatory promotion gate.

## Read-back placement

**Direction.** Both. MCP and SDK `recall` calls are pull. Hosted chat, `/agent/query`, and `buildAndGenerate` are push from the receiving model's perspective: the route/service uses the incoming query/task to retrieve memory and inserts selected memory context before generation ([packages/brain/src/mcp/server.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/mcp/server.ts), [packages/brain/src/sdk/cortex.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/sdk/cortex.ts), [packages/brain/src/services/response.service.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/services/response.service.ts), [apps/server/src/routes/chat.routes.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/apps/server/src/routes/chat.routes.ts)).

**Targeting and signal.** The strongest engineered memory push is `targeting: instance`: every authenticated hosted-chat message starts recall from the user content, skips LLM expansion for latency, recalls up to 25 candidates, optionally adds temporal retrieval when constraints are detected, then trims what is sent. The main signal is `inferred` and mixed across `embedding` similarity, fragment search, `lexical` keyword/BM25 scoring, query-derived entity/graph expansion, temporal matching, and optional reranking. Identifier signals also narrow the pool through owner wallet, related user or wallet, memory type, tags, source exclusions, and min-decay/min-importance filters; the final selector is identifier narrowing plus inferred ranking. `buildAndGenerate` has the same instance-targeted capability when the caller supplies memory options, and it additionally adds a `targeting: coarse` procedural-strategy push by fetching top procedural memories without a query.

**Timing relative to action.** Chat and response-service read-back happens before generation, so it can change the next response. The `/greet` path pushes a non-LLM recap from important memories before the user asks a new substantive question. Dream/reflection outputs are after-action until later recall pushes them back into a future prompt.

**Selection, scope, and complexity.** Selection is code-grounded: owner scope, max results, min decay, source exclusions, token-budget fitting, memory type grouping, temporal constraints, and query-dependent graph expansion. The complexity of loaded context is nontrivial because a single prompt may contain semantic knowledge, procedural rules, identity memories, recent episodes, and conversation history. Effective context dilution and precision/recall are runtime qualities not verified from code.

**Authority at consumption.** In MCP mode, recalled memories are advisory context unless the host prompt tells the model to obey them. In hosted chat, `buildSystemPrompt` presents memories as what Clude remembers and places procedural memories under `<behaviors>`. `formatMemoryContext` goes further for procedural memories by saying the model must follow learned strategies. That is system-definition authority at consumption, even though the source artifact is a memory row ([packages/brain/src/memory/memory.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/packages/brain/src/memory/memory.ts), [apps/server/src/routes/chat.routes.ts](https://github.com/sebbsssss/cludebot/blob/f3cd03c424110a5a506abc7c13d7945f7fd26755/apps/server/src/routes/chat.routes.ts)).

**Faithfulness.** The code proves retrieval and injection, not that the model obeys the retrieved memory. There are benchmark and proof routes for hallucination and token savings, and a confidence-gate experiment can add hedging instructions, but this review did not find a checked-in behavioral ablation proving that a fired memory changed the response in the intended direction.

**Other consumers.** Human users consume dashboard timelines, brain/entity views, memory packs, exported prompts, and trace/explain routes. The hosted API consumes owner-scoped memories. Agents consume MCP tools and injected instructions. Dream/reflection jobs consume memories as source material for new memories.

## Curiosity Pass

Clude has three local/hosted identities that can be easy to conflate: the default MCP path is SQLite at `~/.clude/brain.db`, explicit local mode is JSON at `~/.clude/memories.json`, and hosted/self-hosted Cortex uses HTTP or Supabase. The README's "local-first" framing is broadly true, but the code path matters for which retrieval and dream features are available.

The memory system is both product substrate and product subject. `AGENTS.md` requires every repo agent to use `clude-memory`, and the repository contains a concrete memory dump. That makes the system unusually self-referential: operational memories about Clude help steer work on Clude.

The retrieval pipeline is ambitious enough that governance becomes the hard problem. Vector fragments, BM25, knowledge seeds, entity co-occurrence, graph traversal, JEPA links, type diversity, source penalties, and confidence gates all have plausible local value. Their combined authority is harder to inspect than a smaller number of explicit context-pack rules.

The action-learning loop is one of the more borrowable parts because it closes an actual behavior loop: action, measured outcome, procedural lesson. The weakness is that the measured domains are currently product-specific, such as X/social outcomes, and the generated rule can gain prompt authority later without a human promotion state.

The on-chain/PMP layer gives memories audit and ownership affordances, but it is orthogonal to memory usefulness. A committed hash can prove a memory payload existed; it does not prove the memory is relevant, true, or safe to inject.

The prompt sync path is a quiet activation surface. It can turn exported memories into always-loaded prompt text for another model, which is useful for portability but bypasses the retrieval/budget controls of the hosted chat path.

## What to Watch

- Whether Clude adds an explicit review or trust state for generated procedural and self-model memories before they are inserted as behavior instructions.
- Whether hosted chat gets a faithfulness benchmark that compares responses with and without retrieved memories, not just retrieval quality or token savings.
- Whether the SQLite MCP path gains parity with the Supabase path for graph, dream, encryption, and trace/explain features, or remains a simpler local memory adapter.
- Whether memory packs become a real inter-agent authority boundary with signatures, provenance, and import policies, rather than just portable context bundles.
- Whether confidence gates move from optional experiment into default prompt assembly for low-evidence answers.
- Whether owner-scoped entity and graph retrieval remain leak-safe as more graph and co-occurrence paths are added.

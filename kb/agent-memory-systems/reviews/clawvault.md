# clawvault

ClawVault, by Versatly, is a TypeScript CLI and OpenClaw memory plugin for local-first agent memory. It stores ordinary memories as Markdown folders, keeps internal indexes and facts under `.clawvault/`, compresses session traces into observation ledgers, routes high-importance observations into typed vault categories, builds graph and search indexes, and can inject retrieved context into OpenClaw prompt construction. The repository's README now explicitly deprecates ClawVault for new OpenClaw deployments in favor of OpenClaw native memory, so this is best read as a historical but implemented ancestor of that first-party path rather than as the recommended current deployment target ([README.md](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/README.md)).

**Repository:** https://github.com/Versatly/clawvault

**Reviewed commit:** [bd702e9cce436bc3065827714cd576e8be20c375](https://github.com/Versatly/clawvault/commit/bd702e9cce436bc3065827714cd576e8be20c375)

## Core Ideas

**The durable user-facing substrate is a Markdown vault, not a service database.** `ClawVault.init` creates category folders such as `rules`, `preferences`, `decisions`, `patterns`, `people`, `projects`, `goals`, `transcripts`, `inbox`, `lessons`, `agents`, `commitments`, `handoffs`, `research`, `tasks`, and `backlog`, plus `ledger/raw`, `ledger/observations`, and `ledger/reflections` for observational memory ([src/lib/vault.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/vault.ts), [src/types.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/types.ts)). The ordinary memory files are inspectable, Obsidian-compatible Markdown with frontmatter. Internal state such as `.clawvault/graph-index.json`, `.clawvault/facts.jsonl`, `.clawvault/embeddings.bin.json`, observer cursors, and maintenance reports is adjacent rather than hidden behind a hosted store.

**Session lifecycle is a first-class memory workflow.** `wake` recovers from a dirty checkpoint, builds a recap from handoffs/projects/commitments, and includes recent high-importance observations; `sleep` writes a handoff, can commit the current Git repository, optionally observes a supplied session transcript, and can run reflection ([src/commands/wake.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/commands/wake.ts), [src/commands/sleep.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/commands/sleep.ts), [bin/register-session-lifecycle-commands.js](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/bin/register-session-lifecycle-commands.js)). This makes context-death recovery a normal read-back path rather than an afterthought.

**Observation is trace compression plus routing.** The observer watches or incrementally reads session files, persists raw message JSONL, buffers messages until threshold, uses an LLM compressor when configured or a deterministic fallback otherwise, writes date-scoped observation Markdown, then routes important observations to categories such as decisions, lessons, people, preferences, commitments, projects, tasks/backlog, and facts ([src/commands/observe.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/commands/observe.ts), [src/observer/observer.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/observer/observer.ts), [src/observer/compressor.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/observer/compressor.ts), [src/observer/router.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/observer/router.ts)). This is the main trace-derived learning loop.

**Facts are a separate structured memory layer.** Write-time and hook-time extraction convert text into `(entity, relation, value, confidence, source, validFrom)` records, store them in `.clawvault/facts.jsonl`, supersede conflicting exclusive relations, and build a simple entity graph JSON for hook-derived facts ([src/lib/fact-extractor.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/fact-extractor.ts), [src/lib/fact-store.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/fact-store.ts), [src/plugin/fact-extractor.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/fact-extractor.ts)). The fact store is not merely displayed. The `context` command injects matching facts with priority boosts for preference and temporal queries ([src/commands/context.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/commands/context.ts)).

**Retrieval combines lexical, semantic, graph, fact, observation, and profile logic.** The default in-process search chunks documents, runs BM25, optionally looks up hosted embeddings, fuses BM25 and semantic ranks with reciprocal-rank-style weights, optionally applies hosted cross-encoder reranking, and can fall back to `qmd` depending on configuration ([src/lib/in-process-search.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/in-process-search.ts), [src/lib/search.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/search.ts), [src/lib/embedding-store.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/embedding-store.ts)). `context` then blends search results with recent daily notes, fact-store matches, recent observations, and graph neighbors, ordering them by context profiles such as default, planning, incident, and handoff, with caps and a token budget fitter ([src/lib/context-profile.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/context-profile.ts), [src/commands/context.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/commands/context.ts)).

**OpenClaw integration gives memory both tool and hook authority.** The plugin registers `memory_search` and `memory_get`, adds a recall mandate in `before_prompt_build`, optionally injects prompt-matched memory context, filters outbound questions through `message_sending`, observes active sessions on heartbeat or before compaction, and performs pre-reset checkpointing or fact extraction when enabled ([src/openclaw-plugin.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/openclaw-plugin.ts), [src/plugin/hooks/before-prompt-build.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/hooks/before-prompt-build.ts), [src/plugin/hooks/message-sending.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/hooks/message-sending.ts), [src/plugin/hooks/observation.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/hooks/observation.ts), [src/plugin/hooks/session-lifecycle.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/hooks/session-lifecycle.ts)). The plugin manifest makes execution and several lifecycle features opt-in, but enables before-prompt recall, message filtering, and communication-protocol enforcement by default in its config schema ([openclaw.plugin.json](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/openclaw.plugin.json)).

**Maintenance is staged, not fully autonomous governance.** `maintain` runs curator, janitor, distiller, and surveyor workers over inbox and vault state. These workers can route inbox items, archive duplicates/stale captures, create distilled fact/decision/lesson files, and write maintenance reports, using LLMs when configured and heuristics otherwise ([src/commands/maintain.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/commands/maintain.ts), [src/lib/maintenance/curator-worker.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/maintenance/curator-worker.ts), [src/lib/maintenance/janitor-worker.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/maintenance/janitor-worker.ts), [src/lib/maintenance/distiller-worker.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/maintenance/distiller-worker.ts), [src/lib/maintenance/surveyor-worker.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/maintenance/surveyor-worker.ts)). The system provides promotion machinery, but trust still mostly comes from source visibility and operator review rather than a hard validation gate.

## Artifact analysis

- **Storage substrate:** `files` — Local filesystem folders under the vault root
- **Representational form:** `mixed` — Markdown prose plus YAML frontmatter and wiki-links

**Markdown vault memory files.** Storage substrate: local filesystem folders under the vault root. Representational form: Markdown prose plus YAML frontmatter and wiki-links. Lineage: authored through CLI commands, live-capture candidates, routed observations, inbox maintenance, or direct human editing; frontmatter records dates and some source metadata, but source offsets are usually not preserved. Behavioral authority: knowledge artifact authority when searched, read, or cited as context; system-definition authority when files under `rules`, `preferences`, `decisions`, or injected rule paths are placed into a future prompt as instructions or policy-like context.

**Observation ledger and raw transcript capture.** Storage substrate: `ledger/raw/<source>/<YYYY>/<MM>/<DD>.jsonl` and `ledger/observations/<YYYY>/<MM>/<DD>.md`, with older observations archived under `ledger/archive/observations` by reflection/archive passes ([src/lib/ledger.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/ledger.ts), [src/observer/observer.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/observer/observer.ts), [src/observer/reflection-service.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/observer/reflection-service.ts)). Representational form: raw JSONL message records and scored Markdown observations with typed tags, confidence, and importance. Lineage: trace-extracted from OpenClaw, Claude, ChatGPT, or opencode session material, with source/session/transcript metadata on raw records and date-level citations in reflections. Behavioral authority: raw records are knowledge artifacts for evidence and audit; observations become higher-authority context when `wake` and `context` read them, and they become source material for category routing, tasks, facts, and reflections.

**Routed category files, backlog/tasks, and weekly reflections.** Storage substrate: category Markdown files, backlog/task files, and `ledger/reflections/<week>.md`. Representational form: prose Markdown plus frontmatter and scored observation lines. Lineage: derived from observation Markdown by router category patterns, task de-duplication, maintenance workers, or weekly reflection promotion. Reflection promotion includes high-importance single observations and repeated medium-importance observations, then cites source observation files ([src/observer/router.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/observer/router.ts), [src/observer/reflection-service.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/observer/reflection-service.ts)). Behavioral authority: knowledge artifact authority as persistent memory; system-definition authority when tasks/backlog change what future agents should do, or when reflected patterns and decisions are injected into future context.

**Fact store and entity graph.** Storage substrate: `.clawvault/facts.jsonl` and, for plugin event extraction, `.clawvault/entity-graph.json`. Representational form: symbolic JSONL facts with prose values and raw evidence snippets; graph nodes/edges are symbolic. Lineage: rule-based, LLM, or hybrid extraction from observation Markdown and hook events; each fact stores source, raw text, validFrom, optional validUntil, and confidence, but source precision is coarse. Behavioral authority: facts are knowledge artifacts when searched as evidence, but they also carry ranking and context-selection authority because preference and temporal queries get fact-specific boosts in `context`.

**Graph and search indexes.** Storage substrate: `.clawvault/graph-index.json`, in-memory BM25 chunks, `.clawvault/embeddings.bin.json`, optional qmd indexes, and generated Obsidian Bases files. Representational form: symbolic indexes plus distributed-parametric embeddings when hosted embeddings are configured. Lineage: generated from Markdown files, frontmatter fields, wiki-links, tags, document mtimes, and embedding model signatures ([src/lib/memory-graph.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/memory-graph.ts), [src/lib/in-process-search.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/in-process-search.ts), [src/lib/embedding-store.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/embedding-store.ts)). Behavioral authority: ranking, routing, and selection authority. These are system-definition artifacts because they determine what evidence reaches the agent under search, context, plugin injection, and message filtering.

**OpenClaw plugin configuration, hooks, tools, and runtime state.** Storage substrate: `openclaw.plugin.json`, installed package code, host OpenClaw plugin runtime, and agent-specific vault paths. Representational form: symbolic JSON config, TypeScript hook logic, prose recall mandates, and formatted context injections. Lineage: authored system integration code plus operator config; runtime state caches startup recovery notices and session recaps ([src/plugin/runtime-state.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/runtime-state.ts), [src/plugin/hooks/before-prompt-build.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/hooks/before-prompt-build.ts), [src/plugin/vault-context-injector.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/vault-context-injector.ts)). Behavioral authority: system-definition authority. The hooks prepend system context, register tools, cancel or rewrite outbound messages, trigger observer cron jobs, and run checkpoint/fact-extraction paths.

**Inbox and maintenance outputs.** Storage substrate: `inbox/*.md`, `inbox/archive`, `inbox/merged`, category outputs, `.clawvault/maintenance/*`, and maintenance state/log files. Representational form: Markdown/frontmatter plus JSON-ish state/log records. Lineage: manual inbox captures, live assistant-response capture, heuristic or LLM worker decisions, and content hashes ([src/lib/inbox.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/inbox.ts), [src/capture/service.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/capture/service.ts), [src/lib/maintenance/types.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/maintenance/types.ts)). Behavioral authority: mostly governance and curation advice until a worker writes a durable category file. Once written, the result joins the same retrieval and injection paths as authored memory.

The promotion path is real but uneven: raw traces can become observations, observations can become routed Markdown, tasks, facts, and weekly reflections, and those artifacts can be searched or injected. What is weaker is promotion governance. There is no central typed review contract that says a trace-derived candidate has become a trusted rule, and fact extraction can be asynchronous or heuristic depending on mode.

## Comparison with Our System

| Dimension | ClawVault | Commonplace |
|---|---|---|
| Current status | Deprecated for new OpenClaw deployments; historical code remains readable | Active methodology KB and framework |
| Main substrate | Local Markdown vault plus `.clawvault` state | Git-tracked Markdown collections, schemas, generated indexes, review artifacts |
| Primary workflow | Agent session memory, wake/sleep, observation, recall, context injection | Source-grounded knowledge artifacts, type contracts, validation, review gates |
| Retrieval | BM25, optional embeddings/rerank, graph neighbors, fact store, profile ordering | Lexical search, indexes, authored links, skills, validation/review reports |
| Trace learning | Session traces compressed into observations, facts, routed files, reflections | Trace-derived behavior mostly mediated through explicit reviews, notes, and instructions |
| Activation | CLI pull plus OpenClaw hooks/tools push relevant context and recall mandates | Mostly deliberate pull through search/indexes/skills; stronger authority lives in instructions and validators |
| Governance | Local inspectability, task de-dup, maintenance workers, some source metadata | Collection contracts, type specs, source-pinned reviews, semantic gates, archive/replacement lifecycle |

The strongest alignment is the local, inspectable file substrate. ClawVault treats memory as Markdown files that humans and agents can inspect, link, and sync. Commonplace does the same, but with stronger typed collection contracts and git-native review discipline.

The largest divergence is authority management. ClawVault optimizes for operational continuity: wake the agent, inject relevant memories, extract facts, update tasks, and keep work moving. Commonplace optimizes for durable methodology claims: source-pinned evidence, explicit type specs, validation, review status, and link semantics. ClawVault's pipeline can move a trace into behavior-changing context quickly; Commonplace is slower but clearer about why a retained artifact deserves authority.

ClawVault is also more productized around host-agent activation. The OpenClaw plugin can place recall policy and retrieved snippets directly into a pre-action system context, and can intervene before outbound messages. Commonplace has skills and instructions, but the normal KB read path remains agent-initiated search and navigation unless a workflow explicitly invokes a skill.

**Read-back:** `both` — The CLI supports pull retrieval through search/recall/context, while OpenClaw hooks push retained session recap and prompt-matched vault snippets into agent context before or during action; the recall mandate and communication protocol are shipped baseline context, not read-back

### Borrowable Ideas

**Observation importance as a routing primitive.** Ready as a workshop pattern. ClawVault's `i>=0.80` structural versus `0.40-0.79` potential split gives a practical bridge from noisy transcripts to promotion decisions. Commonplace could use a similar score only as triage metadata, not as trust.

**Separate raw ledger, distilled observations, and routed artifacts.** Ready now conceptually. ClawVault's raw JSONL, observation Markdown, category files, facts, and reflections are useful lineage layers. Commonplace trace workflows should preserve the same raw-to-distilled boundaries when reviewing agent sessions.

**Context profiles for different work modes.** Worth borrowing when Commonplace has runtime context packs. ClawVault's planning/incident/handoff profiles reorder facts, observations, search hits, graph neighbors, and daily notes. Commonplace could express context pack policies declaratively, but should wait for a concrete consumer.

**Before-prompt retrieval as an activation surface.** Needs a use case and careful budget policy. ClawVault demonstrates a compact engineered push path: use the current prompt as the query, cap results, and inject short snippets. Commonplace could benefit from this for narrow workflows, but generic automatic injection risks context dilution.

**Outbound question filtering against memory.** Interesting but not ready. ClawVault checks outgoing questions, searches memory, rewrites with evidence, and may cancel if the message is still a question ([src/plugin/hooks/message-sending.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/hooks/message-sending.ts), [src/plugin/communication-protocol.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/communication-protocol.ts)). Commonplace would need high precision before letting memory block communication.

**Do not borrow authority spread without review gates.** ClawVault's authority lives across Markdown, JSONL facts, generated indexes, plugin config, runtime hooks, maintenance outputs, and LLM extraction prompts. Commonplace should keep stronger distinctions between knowledge artifacts and system-definition artifacts before allowing trace-derived material to guide future agents.

## Trace-derived learning placement

**Trace source.** ClawVault qualifies as trace-derived learning. It reads session files through `observe --compress`, watchers, active OpenClaw session discovery, `sleep --session-transcript`, and plugin heartbeat/compaction hooks. Raw captured records include source labels, session keys, transcript ids, timestamps, and message text in date-scoped JSONL ledgers ([src/commands/observe.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/commands/observe.ts), [src/observer/active-session-observer.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/observer/active-session-observer.ts), [src/observer/watcher.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/observer/watcher.ts)).

**Extraction.** Extraction has several paths. The observer sanitizes and compresses conversation messages into typed observation lines with confidence and importance. The router classifies observations, extracts tasks, writes category records, and extracts facts through rule, LLM, or hybrid modes. Hook fact extraction scans event payloads for text fragments and writes rule-based facts. Live capture extracts `<memory_note>` tags and heuristic assistant-response memories, then quality-gates and stores them ([src/observer/compressor.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/observer/compressor.ts), [src/observer/router.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/observer/router.ts), [src/plugin/fact-extractor.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/fact-extractor.ts), [src/capture/extractor.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/capture/extractor.ts), [src/capture/service.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/capture/service.ts)).

**Scope and timing.** Scope is vault-local and optionally agent-scoped through OpenClaw `agentVaults`. Timing ranges from online-ish watcher/heartbeat/compaction observation, to session-end sleep observation, to explicit `reflect` and `maintain` passes. Weekly reflection promotes stable observation patterns and archives older observations after promotion.

**Survey placement.** On the trace-derived learning survey, ClawVault belongs in the trace-to-observation and trace-to-operational-memory family. It strengthens the claim that durable behavior change usually happens after a distillation boundary: raw session traces are evidence, but the behavior-shaping artifacts are observation lines, routed Markdown files, facts, tasks, reflections, search indexes, and hook-injected snippets.

**Curation policy.** The oracle is mixed. LLM compressors and LLM maintenance workers can supply semantic judgment, but deterministic fallbacks, regex extractors, category patterns, score thresholds, duplicate checks, and operator-visible files are central. This makes the system resilient without API keys, but quality depends on the extraction path actually used.

## Read-back placement

**Direction.** Both. From the agent's perspective, `search`, `vsearch`, `recall`, `context`, `memory_search`, and `memory_get` are pull. The OpenClaw `before_prompt_build` path can push retained memory by prepending current-prompt vault matches and a session recap cache; `message_sending` can push memory-derived evidence into an outbound message or cancel it after a search. The recall mandate, recovery instruction text, and communication protocol appendix are shipped or generated baseline/system surfaces rather than memory read-back. Observation, reset, compaction, and heartbeat hooks write or refresh memory for later consumption, but they are not themselves read-back unless their outputs later feed prompt injection.

**Targeting and signal.** The memory push is instance-targeted. For prompt-building injection, the main signal is `inferred`: the current prompt is sanitized and used as a content query, then `ClawVault.find` searches vault Markdown with BM25 lexical matching by default, optional embedding ranks, optional cross-encoder reranking, a `minScore`, top-k clamp, and temporal boost ([src/plugin/hooks/before-prompt-build.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/hooks/before-prompt-build.ts), [src/plugin/vault-context-injector.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/vault-context-injector.ts), [src/lib/in-process-search.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/in-process-search.ts)). In the model's terms this is `inferred / lexical` by default, with optional `embedding` and reranker `judgment` components when configured. Session recap injection is also instance-targeted, but by `identifier`: the hook resolves the current `sessionKey`/agent id and fetches recent turns for that session. Message filtering is another `inferred / lexical`-first path: if outbound content contains a question, ClawVault searches memory using that content and can rewrite or cancel based on hits over a score threshold.

**Timing relative to action.** Prompt-building injection fires before the model sees the task, so it can change the next action. Message-sending filtering fires before the agent's outbound message leaves the system, so it can prevent a question that memory could answer. Observation hooks and reflection are after-action or maintenance paths unless their outputs later feed prompt injection.

**Selection, scope, and complexity.** Selection is code-grounded: prompt text is sanitized and truncated, max results are clamped to 1-20, default min score is 0.2 for context injection, snippets are truncated, and vault path can be resolved per agent or workspace. Session recap injection is scoped by session key and limited to the recent recap entries. The CLI `context` command has a richer pull selection model with profiles, source caps, graph-hop expansion, fact boosts, and optional token budgets. Effective precision, recall, and context dilution are runtime qualities not verified from source.

**Authority at consumption.** Injected memories are advisory context, but the recall mandate and communication protocol are system-definition artifacts because they tell the agent what to do while not themselves being read-back. The message-sending hook has stronger gate authority because it can cancel an outbound question after a memory search. `memory_get` is intentionally scoped to `MEMORY.md` and `memory/*`, which narrows tool authority compared with arbitrary file reads ([src/plugin/memory-manager.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/memory-manager.ts)).

**Faithfulness.** The code wires retrieval and injection, but it does not prove that injected memory changed model behavior. The repository includes benchmark/eval directories, but this review did not find a checked-in ablation analogous to "with injected memory versus without injected memory" for the OpenClaw hook path.

**Other consumers.** Human users consume the Markdown vault, Obsidian views, dashboard, maintenance reports, and CLI outputs. Maintenance workers consume inbox files and vault surveys. The OpenClaw host consumes plugin config and hook results. Search indexes and graph indexes consume the same Markdown substrate as ranking/routing inputs.

## Curiosity Pass

The deprecation notice is unusually important. ClawVault has substantial implementation, but the README tells new OpenClaw users to choose OpenClaw native memory. That makes this review more useful as design archaeology than as an adoption recommendation.

The plugin config is security-aware. Child-process execution is gated by `allowClawvaultExec`, optional binary path and SHA-256 checks exist, and environment access is separately gated; direct in-process library retrieval is still used for before-prompt context injection ([openclaw.plugin.json](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/openclaw.plugin.json), [src/plugin/clawvault-cli.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/clawvault-cli.ts), [src/plugin/integrity.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/plugin/integrity.ts)).

The fact extraction `llm` and `hybrid` router paths start asynchronous writes and return before the LLM result is necessarily stored. That keeps routing non-blocking, but it means immediate summaries can under-report facts and downstream context may lag extraction ([src/observer/router.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/observer/router.ts)).

The graph layer is more structural than semantic. It indexes wiki-links, tags, and selected frontmatter relation fields; graph expansion in `context` is useful local navigation, not a general knowledge-graph reasoner ([src/lib/memory-graph.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/lib/memory-graph.ts), [src/commands/context.ts](https://github.com/Versatly/clawvault/blob/bd702e9cce436bc3065827714cd576e8be20c375/src/commands/context.ts)).

The system has two different "fact graph" stories: the Markdown memory graph and the entity graph derived from extracted facts. They share a graph vocabulary but are built from different substrates and have different authority.

The `sleep` command's optional Git commit uses `git add -A` inside the reviewed system. That is a reasonable product feature for its own users, but it is exactly the kind of broad staging policy Commonplace avoids in shared agent worktrees.

## What to Watch

- Whether ClawVault remains only a deprecated bridge or whether code from this repository continues migrating into OpenClaw native memory.
- Whether OpenClaw native memory preserves ClawVault's raw trace, observation, routed artifact, fact, and reflection layers, or collapses them behind a narrower memory API.
- Whether before-prompt injection gains an explicit faithfulness benchmark or ablation showing behavior change rather than context presence.
- Whether fact extraction becomes synchronous, queued, or status-visible so downstream context freshness is easier to reason about.
- Whether trace-derived routed files gain stronger source citations, offsets, and review states before they function as rules or commitments.
- Whether the maintenance workers become governed promotion paths or remain useful hygiene and triage helpers.

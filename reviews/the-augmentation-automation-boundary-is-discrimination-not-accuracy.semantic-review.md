=== SEMANTIC REVIEW: the-augmentation-automation-boundary-is-discrimination-not-accuracy.md ===

Claims identified: 16

WARN:
- [Grounding alignment] The note states "calibration improves but discrimination does not" and calls discrimination "empirically stagnant," presenting this as a settled finding from Rabanser et al. The source is more nuanced: "discrimination trends diverge across benchmarks: on tau-bench it has generally improved in recent models, whereas on GAIA it has in fact mostly worsened." The source finding is benchmark-dependent and mixed, not a clean stagnation result. The note's phrasing ("Rabanser et al. confirm this empirically") overstates the uniformity of the evidence. The open questions section partially acknowledges uncertainty ("the stagnation finding may be temporary") but frames this as a scale question, not a benchmark-dependence question the source itself raises.

- [Completeness / boundary case] The note presents two routes to automation: (a) self-assessment via confidence scores, (b) external hard oracles. Boundary case: **a human-designed heuristic filter that is neither model self-assessment nor a hard oracle** — e.g., a learned classifier trained on model internals (probing), or an ensemble disagreement signal. This is not self-assessment in the confidence-score sense (the model does not rate itself), nor is it an external hard oracle (it is probabilistic). It falls between routes (a) and (b). The note's framing compresses a spectrum into a binary, which strains when applied to soft external oracles. The three-part taxonomy at the end partially addresses this (it includes "self-assessment + high discrimination" as a middle case), but the two-route framing in the body treats self-assessment and external oracles as exhaustive.

- [Grounding alignment / scope mismatch] The note claims "scaling model capability reliably improves aggregate accuracy" as a general fact. Rabanser et al. do find accuracy improvements over 18 months of releases, but the source also notes "reliability does not scale uniformly with model capability" and the improvements are benchmark-specific. The note promotes this to a general scaling law ("reliably improves") that the source does not assert at that level of generality. The contrast the note builds — accuracy scales, discrimination does not — is stronger than the source warrants.

INFO:
- [Completeness / boundary case] The note claims "augmentation requires only aggregate accuracy." Boundary case: **augmentation where human review is perfunctory or absent in practice** — e.g., a coding assistant whose suggestions are accepted without reading, or a search copilot whose summaries are trusted without checking sources. In these cases the human nominally provides discrimination but actually does not, making the system function as de facto automation despite being architecturally augmentation. The note's clean split between augmentation (human catches errors) and automation (nobody catches errors) does not account for this degraded-augmentation regime, which is arguably where many real deployments sit.

- [Completeness / boundary case] The three-part taxonomy (external hard oracle / self-assessment + high discrimination / self-assessment + low discrimination) omits a case: **external soft oracle available, low self-assessment discrimination**. For example, an LLM-as-judge setup where the judge has moderate but not hard discriminative power. This falls between the "hard oracle" row and the "only self-assessment" rows. The oracle-strength-spectrum note that the reviewed note links to addresses this gradient explicitly, but the taxonomy here collapses soft external oracles into the hard-oracle category.

- [Internal consistency] The note defines discrimination as "the ability to know, per instance, whether this particular output is likely wrong" (opening paragraph) and later as "per-instance separation — whether the model assigns higher confidence to its correct outputs than to its incorrect ones" (Discrimination section). These are subtly different: the first is about the system having self-knowledge of error likelihood; the second is a statistical property of confidence-score distributions. The second definition (confidence-score separation) is more precise and is what Rabanser et al. actually measure. But the first definition is broader — a system could "know when it is wrong" through mechanisms other than confidence scores (e.g., internal anomaly detection). The note generally uses the narrower confidence-score sense, which is appropriate for the Rabanser evidence, but the opening paragraph's broader framing slightly overstates what the evidence actually addresses.

- [Grounding alignment] The note cites MAKER as achieving "zero errors over a million steps precisely because every sub-task has a deterministic oracle — the system never needs the model to know when it is wrong." The MAKER ingest confirms this characterization is accurate. However, the note uses MAKER to exemplify route (b) for automation generally, while the MAKER ingest's own limitations section notes that Towers of Hanoi is an ideal case (single correct move per step, compact lossless state, known decomposition structure) and that extension to real-world domains with ambiguous per-step correctness is undemonstrated. The note does not acknowledge this limitation when citing MAKER as proof that route (b) works.

- [Internal consistency] The note's title claim — "the augmentation-automation boundary is discrimination not accuracy" — is a crisp binary framing. But the body develops a more nuanced position: the boundary is "not a fixed accuracy threshold but a function of available oracle strength." These are compatible (the boundary is about oracle strength, of which discrimination is one dimension), but the title suggests discrimination is THE factor, while the body shows it is one factor alongside external oracle availability. The body's analysis is more precise than its title.

Section 3.3 defines calibration (P_cal) and discrimination (P_AUROC) as separate metrics, and the predictability examples in Appendix A.1.3 give a concrete case where perfect calibration coexists with zero discrimination. The note's "perfectly calibrated yet assign the same confidence to every output" example mirrors this.

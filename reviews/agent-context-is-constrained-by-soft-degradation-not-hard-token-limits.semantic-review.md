=== SEMANTIC REVIEW: agent-context-is-constrained-by-soft-degradation-not-hard-token-limits.md ===

Claims identified: 19

WARN:
- [completeness] The note claims the soft bound operates across exactly "two dimensions" — volume and complexity — framing this as a complete decomposition. But a boundary case exists that fits neither cleanly: **information arrangement** (position effects). The "lost in the middle" finding is cited under volume, but position effects are not about how many tokens there are — they occur at fixed token counts by rearranging content. The note's own Benchmarker subsection acknowledges "information arrangement" as a factor that shifts the soft bound, but the two-dimensions framework has no slot for it. Position effects could be forced into "complexity" (arrangement changes how hard the tokens are to use), but the note defines complexity as "compositional depth" and "interpretation overhead," not spatial layout. This is a gap in the enumeration: a factor the note itself names as relevant but that the framework doesn't cleanly accommodate.

- [grounding] The note claims "ConvexBench shows complexity-driven collapse at low token counts — compositional depth, not volume, was the bottleneck." The source (Liu et al., 2026) studies whether LLMs can recognize convexity in deeply composed symbolic functions — a formal mathematical verification task with mechanically verifiable answers. The note uses this to ground a general claim about LLM context complexity, but ConvexBench measures compositional reasoning over symbolic expressions (parenthesized function compositions), not the kind of complexity practitioners encounter in agent contexts (competing instructions, scope leakage, underspecified natural-language tasks). The domain of the source (symbolic mathematical verification) is narrower than the domain of the claim (general agent context complexity). The transfer is plausible but not established by the source itself.

- [grounding] The note characterizes Paulsen's MECW work as confirming that "usable context can be far below advertised windows and is task-dependent." This is accurate as far as it goes, but the note places this finding under the "Volume" dimension, implicitly treating MECW as a volume phenomenon. The Paulsen source itself acknowledges a limitation: the tasks used (counting, sorting, filtering, summing) are themselves computationally demanding for LLMs — the study "focuses on one variable — token count" but the tasks confound volume with task difficulty. The note's own sibling (effective-context-is-task-relative) notes this confound explicitly: "Paulsen does not cleanly isolate pure volume from task difficulty (counting, sorting, and filtering are themselves LLM-hard)." By placing Paulsen purely under "Volume" without noting this confound, the note presents a cleaner attribution than the source supports.

INFO:
- [completeness] The note's most extreme boundary case — what happens when an agent is given a very small context (say 50 tokens of a single clear instruction) — is not addressed. The note frames the soft bound as a degradation curve, but doesn't clarify whether there is a minimum context below which the soft bound is not operative (i.e., where quality is simply determined by instruction clarity, not degradation). This would clarify whether the framework applies universally or only above some complexity/volume floor.

- [completeness] The "programmatic constructability" claim in the Consequences section introduces a new concept — "high control over inputs, low observability of effective processing" — that functions as a third property of the soft bound (alongside the two dimensions and invisibility), but it is presented only as a practitioner consequence rather than integrated into the framework. It could be read as a consequence of invisibility (which it is) or as a distinct structural feature. The note leaves this ambiguous.

- [internal-consistency] The Volume section says "More tokens dilute attention" and cites the "lost in the middle" finding as evidence. But "lost in the middle" is a position effect (middle tokens get less attention than beginning/end tokens), not a pure dilution effect (all tokens getting proportionally less attention). These are different mechanisms: dilution predicts uniform degradation as volume grows; primacy/recency bias predicts positional degradation regardless of volume. The note conflates them under the same heading without distinguishing the mechanisms.

- [grounding] The note states context windows have grown "roughly 30x per year since mid-2023." This is a factual claim without a citation in the note itself, though the context-efficiency sibling note cites Epoch AI (2025) for this figure. The claim is plausible for the 2023-2025 period (8K to 128K to 1M+), but "30x per year" as a continuing rate would be extraordinary and possibly misleading if read as a forward projection rather than a historical observation.

The note's use of the term and its characterization align with this source.

The indirection note explicitly describes interpretation overhead as a per-read cost that compounds with layers of abstraction — this is a clean match for the complexity dimension as the note uses it.

The relationship semantics ("sharpens") are appropriate.

The Consequences section does not backtrack or soften this into "both matter equally" — it maintains the asymmetry while acknowledging the hard limit exists. The note is internally coherent on its central thesis.

The Interaction subsection appropriately qualifies that they are "distinguishable but not fully separable" rather than claiming full independence. No definition drift was detected for "volume" or "complexity" across their uses.

Each level describes a different manifestation of the same property (unobservability) without contradicting the others. The progression from operational to epistemic to commercial is coherent.

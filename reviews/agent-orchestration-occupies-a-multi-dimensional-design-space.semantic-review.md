WARN:
- [Completeness] The five dimensions do not include an **authority/delegation** dimension. The linked note "agent orchestration needs coordination guarantees, not just coordination channels" identifies a fourth failure mode -- accountability vacuum in delegation chains -- that is not a coordination guarantee in the same sense as the other three (it is a governance failure, not a semantic one, as that note explicitly says). The guarantee families enumerated in this note (isolation, consistency, adjudication) cover shared-substrate failures but omit delegation-chain governance. Since the guarantees note itself treats accountability vacuum as a distinct row in its table, the three-family enumeration here undercounts its own source. This may be intentional simplification (the note says "three main guarantee families") but the word "main" obscures a qualitative omission, not a minor variant.
- [Completeness] The **coordination form** enumeration lists five forms but omits **output aggregation / synthesis** as a distinct form. The guarantees note treats "output aggregation" as one of four composition modes (flat context, shared memory, output aggregation, delegation chain). If output aggregation is important enough to have its own failure mode (amplification) and its own missing primitive (adjudication), it seems important enough to be a coordination form. The current list could arguably absorb it under "shared-state / blackboard" but the fit is strained -- aggregation is about combining finished outputs, not sharing state during execution.
- [Grounding] The note claims "Forking changes coordination form without changing scheduler placement" and cites the voooooogel ingest source. The source does argue forking is structural and discusses it as a scoping/spawning primitive. However, forking in the source's description does implicitly affect scheduler placement: a forked instance carries the parent's full context and operates independently, meaning the fork decision itself is a scheduling decision about how to distribute work. The note's claim that forking varies *only* coordination form is a simplification the source doesn't explicitly make. The source treats forking as a more fundamental primitive that cuts across multiple concerns.

INFO:
- [Completeness] The **persistence horizon** enumeration uses three levels (per-step, per-session, cross-session) but the boundaries between levels are underspecified. Where does a system that persists state for a fixed number of steps (e.g., a sliding window of the last N episodes) fall? It is not per-step (it survives multiple steps) nor per-session (it may not survive the full session). This is a gradient, and the note implicitly acknowledges this by calling Slate "an intermediate point," but the three named levels suggest a discrete taxonomy rather than a continuum.
- [Completeness] The **boundary-return artifact** dimension lists four types but does not mention **tool-call requests** as a return artifact. The bounded-context orchestration model note explicitly discusses tool calling as "a particular return shape from a bounded call" that is distinct from prose, structured extraction, or code. Tool-call requests could be folded into "structured extraction" but they have a distinctive property (they request further action rather than delivering a result) that the other four types lack.
- [Completeness] The five dimensions are claimed to be "separable" but the note does not discuss which pairs interact. The "Why this matters" section mentions that Slate changes two dimensions simultaneously, which is evidence of interaction, not independence. The note's open question "Which dimensions interact, and which vary independently?" acknowledges this gap, but the framing "separable dimensions" in the introduction may overstate what has been established.
- [Grounding] The Slate ingest source describes episodes as "compositionally reusable -- one thread can be initialized with another thread's episode." The note summarizes Slate as changing "persistence horizon and boundary-return artifact simultaneously." This is accurate but incomplete -- the ingest source also emphasizes that Slate changes coordination form (episodes as a new coordination primitive beyond conversation/refinement/forking, per the ingest's connection to the conversation-vs-prompt-refinement note). The note's explanation of why Slate breaks one-axis taxonomies understates how many dimensions Slate actually varies.

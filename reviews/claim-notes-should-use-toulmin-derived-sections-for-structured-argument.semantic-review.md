WARN:
- [Grounding alignment — affordance table] The note claims "Types mark affordances lists `claim` affordances as: verify, gather evidence, challenge, use as assumption" and attributes this to the linked note `instructions-are-typed-callables.md`. That note does not contain an affordance table with those four operations. It mentions that "a `structured-claim` affords verification" in passing and has a skill-signature table (ingest, connect, validate, convert), but no enumeration of claim-specific affordances. The four operations (verify, gather evidence, challenge, use as assumption) appear to be the reviewed note's own inference, not an attribution from the source. This makes one of the "three independent threads" of evidence weaker than presented — the thread is the note's own analysis, not something the source arrived at independently.
- [Grounding alignment — link semantics as warrants] The note states "Link semantics using 'since' and 'because' encode warrants" (Evidence item 1), attributing this to the title-as-claim note. The title-as-claim note describes "since" and "because" as link semantics that encode causal and foundational relationships, and separately in its Relevant Notes section describes Toulmin as "the theory behind what this note describes." However, the title-as-claim note's body text never uses the term "warrant" to describe what "since"/"because" encode — the warrant framing is only in the footer link annotation, which was likely written after this structured-claim note existed. The Toulmin mapping (link semantics = warrants) may be more the reviewed note's retroactive reading than something the title-as-claim note itself arrived at independently, weakening the "three independent threads" convergence claim.
- [Completeness — boundary case: implicit-warrant notes] The note advocates for an Evidence section that separates "checkable fact" from reasoning. Boundary case: many KB notes derive their claims from architectural reasoning or conceptual analysis rather than checkable observations (e.g., "If the type were called `claim`, an LLM reading 'titles should be claims' would infer every note needs `type: claim`" — this note's own reasoning is a thought experiment, not checkable evidence). When Evidence sections are filled with reasoning-from-principles rather than verifiable observations, the Evidence/Reasoning split collapses. The note's own Caveats section partly acknowledges this with the "flow problems" caveat, but does not address the more specific issue that the Evidence/Reasoning distinction may not carve nature at its joints for conceptual-argument notes.

INFO:
- [Completeness — boundary case: notes with mixed argument shapes] The note assumes structured-claim notes contain a single top-level argument with one claim (the title), evidence for that claim, and reasoning connecting them. Boundary case: notes that make a primary claim supported by multiple sub-arguments, each with their own evidence and warrants. The current template has flat Evidence and Reasoning sections with no nesting. A note arguing "X because (A because a1, a2) and (B because b1, b2)" would need to interleave evidence and reasoning rather than separating them into distinct sections. The note does not address whether the template accommodates compound arguments or whether those should be decomposed into separate notes.

The body and caveats are not contradictory (the note qualifies the type as experimental), but the tension is strong enough that a reader could reasonably conclude the experiment has already produced negative results.

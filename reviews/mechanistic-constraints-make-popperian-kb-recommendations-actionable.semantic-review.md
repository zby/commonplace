WARN:
- [Completeness] The three proposed practices are presented as derived from KB mechanics, but the derivation path is uneven. Practices 1 (falsifier blocks) and 2 (contradiction-first connection) follow directly from "criticism must be structural" plus "bounded context prevents ambient detection." Practice 3 (rejected-interpretation capture) follows from a different line of argument — underspecification and interpretation error — and its connection to bounded context is weaker. The note says "This is constraining driven by error, and across sessions it compounds as deploy-time learning," but the mechanism of capturing rejected interpretations would be equally valuable in an unbounded-context system. If a system could load everything, rejected interpretations would still need to be recorded somewhere durable. This means practice 3 is motivated more by statelessness (information lost between sessions) than by bounded context or the structural-criticism argument. The note's framing implies all three are forced by the same mechanics, but the third stands on different ground.

- [Completeness] The claim-vs-interpretation two-level distinction (claim 2) is presented as exhaustive — "Both are criticizable, but they live at different levels and need different criticism mechanisms." A boundary case probes this: what about structural or architectural conjectures? When an agent decides to organize content into a particular directory structure, or to split vs. merge notes, that is neither a propositional claim in a note nor a runtime interpretation of an instruction. It is a design-level conjecture about knowledge organization. The note's two levels (claim and interpretation) do not clearly cover this case. A third level — structural/organizational conjectures about KB architecture — might belong here, or might be subsumable under one of the two, but the note does not address it.

- [Grounding] The note claims "Contradictions between notes loaded into the same context aren't flagged; they're silently averaged." The linked source (llm-context-is-composed-without-scoping.md) discusses flat concatenation, spooky action at a distance, name collision, and inability to reason locally. It does not specifically claim that contradictions are "silently averaged" — that is the reviewed note's own characterization. The source's framing is about scope contamination and interference, not about a specific averaging mechanism for contradictions. The inference is plausible (interference could manifest as averaging), but "silently averaged" is a stronger and more specific claim than the source supports. The source says everything is global and there's no isolation; it does not characterize what happens when contradictory content coexists.

INFO:
- [Completeness] The note's own falsifier section identifies one defeat condition: "If bounded-context systems could reliably detect contradictions without externalized structure." A second defeat condition is left implicit: if LLM self-review under the same prompt were NOT actually correlated (i.e., if naive re-reading reliably found flaws), then the "criticism must be structural" argument would also weaken — the problem would be solved by just asking the model to re-check. The error-correction source note does discuss decorrelation requirements, so the claim is well-grounded, but the falsifier section could be more complete by stating both conditions.

- [Grounding] The note says falsifier blocks "convert criticism from 're-read and see if it still seems right' (correlated, weak) into 'check whether this specific condition has been observed' (decorrelated, testable)." The error-correction source note (error-correction-works-above-chance-oracles-with-decorrelated-checks.md) supports the decorrelation framework thoroughly, but does not specifically discuss falsifier blocks as a decorrelation mechanism. The connection — that checking a specific stated condition is more decorrelated than general re-reading — is a reasonable inference the reviewed note constructs on top of the source, not something the source itself argues. The inference is sound but worth flagging as the note's own contribution rather than something grounded in the linked source.

- [Completeness] The simplest possible instance of "a claim that needs a falsifier" would be a definitional note — e.g., "codification is committing a procedure to a symbolic medium." The note partially addresses this: "Not every claim has a clean falsifier, and that's fine. The value is in the attempt, not universal coverage." It also mentions "definitional? tautological? too vague to test?" as things the attempt reveals. This is adequate, but the boundary between "claims that benefit from falsifiers" and "definitions that don't" is left to the writer's judgment without guidance. For a note that proposes a concrete practice (add falsifier blocks to WRITING.md templates), this gap matters — the template will need to say when to skip the block.

- [Internal consistency] The note's title claims that KB mechanics "make Popperian KB recommendations actionable." The body derives the practices from bounded context and underspecification. But the note also acknowledges that claim titles and link semantics "already do part of this." If existing conventions already externalize conjectures and structure criticism, the note's claim is specifically about what the current system STILL lacks (falsifiers and contradiction-first passes), not about making Popperian recommendations actionable in general. The title slightly oversells — it implies the mechanistic argument is what makes Popper actionable, but part of the actionability was already present before this analysis.

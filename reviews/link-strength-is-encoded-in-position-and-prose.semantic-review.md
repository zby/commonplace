WARN:
- [completeness] The five-level position taxonomy has a gap between "inline, parenthetical mention" (Medium) and "footer with context phrase" (Weak). An index entry with a context phrase — e.g., an area index line like "- [note](./note.md) — extends this by adding the temporal dimension" — is neither inline prose nor a footer "Relevant Notes" section. Index entries with context phrases are a significant link type in this KB (the WRITING.md checklist requires them), and they could plausibly be Medium or Weak. The taxonomy is document-scoped (it classifies positions within a single note) but link strength in a KB is graph-scoped — the same note can be linked from many different positions in many different documents, including indexes that are neither inline prose nor footer sections.

- [grounding] The note claims the link contracts framework "treats all links as equal edges." The link contracts framework note actually defines a "link intent taxonomy" with nine distinct intents (Definition, How-to, Reference, Example, Rationale, Evidence, Tool, Related, Index/Hub) and differentiates between inline links and annotated links. It also distinguishes optional vs required links ("If something is required to proceed, summarize it — don't outsource to a link"). While it does not use the word "strength" or assign weights, it does not treat all links as equal — it distinguishes links by intent and by how critical they are to the reader. The note's framing positions itself as adding a dimension the framework lacks, but the framework already has a partial version of that dimension via intent types and the optional/required distinction.

- [grounding] The note claims link-weighted scoring "prevents the credibility erosion problem" and cites arscontexta.md. The arscontexta note describes credibility erosion as a failure mode of *embedding-based* linking systems where unvetted auto-generated links erode trust in all links. The remedy arscontexta describes is curated links with articulated reasons (replacing embedding-based links), not link-weighted scoring of existing links. The note silently extends the arscontexta concept: arscontexta says "replace bad links with good links," while this note says "weight existing links by strength so bad ones don't inflate scores." These are different mechanisms addressing related but distinct problems. Readers following the citation may expect the source to discuss weighted scoring, but it does not.

INFO:
- [completeness] The two strength taxonomies (position-based and prose-based) are presented as independent axes but not combined into a single framework. A link could be in a strong position (inline premise) but use weak prose ("see also [X]"), or in a weak position (footer) but with strong relationship articulation ("contradicts the central claim of [X]"). The note does not address how these axes interact or which takes precedence when they conflict. The load-bearing test (claim 5) implicitly resolves this in favor of position, but this is not stated.

- [completeness] The prose pattern taxonomy (table 2) includes "contradicts" / "but see" as Strong (different direction) but does not account for qualification patterns like "unless [X]", "except when [Y]", or "with the caveat that [Z]." These are common in argumentative prose and indicate a note that constrains or bounds the current note's claim — arguably a distinct role from contradiction ("tension") and from premise ("since"). They could be Strong or Medium depending on framing.

- [grounding] The note says link strength is "the practical answer to the navigation decision problem" and cites agents-navigate-by-deciding-what-to-read-next.md. That note discusses context carried by pointers as the key to navigation decisions — how much information a pointer provides about its target. This is related to but distinct from link strength as defined here. A footer link with a rich context phrase (which this note rates as Weak) could carry more navigational context than an inline premise link to a note whose title is unclear. The navigation note's concept of "context at the point of decision" does not map directly onto the strength hierarchy this note proposes.

- [internal consistency] The note recommends the "inferred approach" for detecting strength, acknowledging that prose analysis is "fuzzy — 'since' might be temporal, not causal." But the entire prose-based taxonomy (table 2) relies on exactly this kind of word-level inference. If "since" is ambiguous between temporal and causal readings, the same ambiguity applies to "extends" (could mean "makes longer" vs "builds upon") and "but see" (could introduce a mild qualification vs a fundamental challenge). The note's recommendation to defer prose analysis to later while already defining the prose taxonomy creates a tension — the taxonomy is presented as authoritative, but its own feasibility section flags the unreliability of the signals it catalogs.

- [completeness] The open questions ask whether link strength should be static or dynamic (property of the link vs link x source note status). This is a significant design question, but the body of the note treats strength as entirely static — the two taxonomies are pure link properties. If strength is dynamic, the entire scoring proposal in the "Note scoring" section needs qualification, because a note's quality score would need to be recomputed whenever any source note's status changes, not just when links are added or removed.

PASS:
- [internal consistency] The five-level position taxonomy and the five-level prose taxonomy are internally consistent — they use the same scale (strongest to weakest) and the examples given for each level are appropriately ordered. The load-bearing test is consistent with the position taxonomy: inline premise links would fail the removal test (paragraph breaks), footer bare links would pass it (paragraph unaffected).
- [grounding] The note-scoring claim ("inbound links by strength... three inline premise links... say more... than twenty footer 'related' links") is consistent with notes-need-quality-scores-to-scale-curation.md, which explicitly states "it must be weighted by link strength — ten footer 'related' links count less than three inline 'since [X]' premise links" and cites this note as the source of the link strength concept. The two notes are mutually consistent and properly cross-referenced.
- [grounding] The quality-signals-for-kb-evaluation.md note discusses PageRank/eigenvector centrality and link articulation quality as signals, which is consistent with the note's claim that link-weighted scoring belongs in graph health measurement. The quality signals note does not explicitly discuss link-strength-weighted PageRank, but the concept is a natural extension of its existing signals.
- [internal consistency] The "Should strength be explicit metadata?" section's conclusion (inferred from position, defer prose analysis) is consistent with the note's own acknowledgment that prose patterns are fuzzy. The recommendation follows logically from the body's evidence.

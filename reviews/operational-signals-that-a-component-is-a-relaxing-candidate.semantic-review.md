WARN:
- [Completeness] The enumeration of six signals implicitly claims to cover the space of operational indicators for detecting vision-feature components. A boundary case that falls between items: **performance degradation under increased concurrency or load**. A component that works correctly at low throughput but breaks under concurrent use may be encoding a theory about execution context (e.g., sequential processing assumptions) rather than specifying the problem. This does not cleanly map to any of the six signals. It is not paraphrase brittleness (the inputs are identical), not an isolation-vs-integration gap (the component may fail even in isolation under load), not process constraints (it may specify outcomes), not hard-to-specify failure (you can specify "must handle N concurrent requests"), and not distribution sensitivity (the input distribution is unchanged). It is closest to Signal 5 but the shift is in operational conditions, not input distribution. Load-sensitivity as a relaxing signal seems plausible and is not covered.
- [Grounding — scope mismatch, Ma et al.] The note claims: "Their finding that smaller models can achieve superior stability (Qwen-1.5B highest AUC-E) supports the interpretation that brittleness detects badly-fitting theories rather than capacity limitations." The ingest file's own Curiosity Gate section challenges this interpretation: "distilled models learn a narrower repertoire of solutions, so there is less variation to express. If the model only knows one way to solve a problem regardless of how you ask, it looks 'stable' without actually being robust in any deep sense." The ingest also notes Ma et al. provide "no causal mechanism for the non-monotonic scaling finding." The note's interpretation (brittleness = badly-fitting theory, not capacity limitation) is one possible reading, but the source is consistent with an alternative reading (stability-through-simplicity, not stability-through-understanding) that would undermine the inference. The note presents the interpretation without acknowledging this ambiguity.
- [Grounding — domain coverage, Rabanser et al.] The note interprets R_prompt findings as "evidence that paraphrase brittleness is a measurable and prevalent signal — exactly what you'd expect if many agent components encode theories about input format rather than specifications of meaning." Rabanser et al.'s study is about agent-level prompt robustness (rephrasing whole task instructions), not about individual component-level theory detection. The source measures whether agents handle semantically equivalent task reformulations, which is about agent behavior, not about identifying which components within a system encode theories vs. specifications. The note's inference — from "agents are brittle to prompt rephrasings" to "this shows components encode theories about input format" — crosses a level of analysis (agent-level measurement to component-level diagnosis). The source does not discuss the bitter lesson boundary, theory/spec distinctions, or component-level relaxing decisions.

INFO:
- [Completeness] Adjacent boundary case: **temporal brittleness** — a component that works correctly now but degrades as the world changes (e.g., a date parser built around current conventions, a sentiment model calibrated to current slang). This overlaps with Signal 5 (distribution sensitivity) but the shift happens over time rather than across data distributions. It could be forced into Signal 5 if you define "distribution" broadly enough to include temporal drift, but the note's framing of Signal 5 ("works on training/development data but degrades on slightly different inputs") suggests spatial rather than temporal distribution shift. Temporal brittleness might deserve explicit mention.
- [Completeness] Degenerate boundary case: a component that passes all five early signals yet the note's opening paragraph explicitly acknowledges this is possible ("a well-regularised learned component can pass all five early signals yet still encode a theory that scale will supersede"). The note is self-aware about this limitation, which is good — but it creates a tension with claim 11 (composition failure as "the strongest evidence"). If the five early signals can all pass for a vision-feature component, and composition failure is only discoverable through expensive integration, the practical value of the early signals for detecting vision features is weaker than the note's structure implies (five early signals presented as the main content, composition failure as an afterthought).
- [Grounding — inference validity] The note states "Neither source connects this to the bitter lesson boundary, but we interpret the findings as evidence that paraphrase brittleness is a measurable and prevalent signal." This is transparent about the interpretive move, which is appropriate. However, the inference has a gap: showing that paraphrase brittleness exists and is measurable does not by itself support the specific claim that brittleness indicates theory-encoding rather than specification-encoding. Both sources measure brittleness as a general property of models, not as a diagnostic for the theory/spec distinction. The note's framing ("exactly what you'd expect if many agent components encode theories about input format") is a consistency argument, not a grounding argument — the evidence is consistent with the theory but does not specifically support it over alternatives.
- [Internal consistency] The note says in the opening that signals "detect *badly-fitting* theories, not the theory/spec distinction itself." But Signal 4 ("Hard to specify failure conditions") and Signal 3 ("Process constraints rather than outcome constraints") seem to target the theory/spec distinction directly rather than detecting badly-fitting theories. If you cannot specify failure conditions, that says something about the nature of the specification, not about how well a theory fits. Similarly, process-vs-outcome constraints is a structural property of the spec, not a measure of theory fit. The opening caveat applies cleanly to Signals 1, 2, 5, and 6 (which detect empirical symptoms of bad fit), but less cleanly to Signals 3 and 4 (which detect structural properties of the spec).

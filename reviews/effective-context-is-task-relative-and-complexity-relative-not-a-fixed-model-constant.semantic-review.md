=== SEMANTIC REVIEW: effective-context-is-task-relative-and-complexity-relative-not-a-fixed-model-constant.md ===

Claims identified: 18

WARN:
- [Grounding alignment / scope mismatch] The note says Paulsen "measures Maximum Effective Context Window (MECW) across 11 frontier models and finds it far smaller than advertised limits. Crucially, the threshold shifts by problem type." The Paulsen source does support this, but the note then uses this as evidence that "effective context is relational" in a way that includes prompt difficulty / compositional complexity as a dimension. Paulsen's tasks (needle-in-a-haystack, counting, sorting, summarization) vary by task *type* but do not isolate compositional complexity as a separate axis. The note's caveats paragraph acknowledges this ("Paulsen does not cleanly isolate pure volume from task difficulty"), but the body paragraphs present Paulsen and ConvexBench as two independent legs of a convergent argument where Paulsen covers the task-type axis and ConvexBench covers the complexity axis. In practice, Paulsen's finding is that MECW varies by task type, not that it varies by complexity independent of task type. The note's synthesis — that effective context is jointly determined by task type *and* complexity — goes beyond what the two sources independently demonstrate, because neither source measures the interaction between the two dimensions. The convergence is weaker than "two independent sources converge on this" suggests: each source covers a different axis, but no source covers the joint claim.

- [Completeness / boundary case] The note claims "effective context is relational: model choice matters, task type matters, and prompt difficulty changes the effective cost of a prompt." This enumeration of factors (model, task type, prompt difficulty) omits several factors that both sources hint at. Paulsen's source discusses **position effects** (retrieval drops from 76% to 66% when key info moves from position 1 to position 2), **ratio of relevant to total tokens** (up to 25% impact), and **training-time sequence length** as distinct factors affecting MECW. The note's caveats mention "position, framing, and scope contamination" but fold them into "prompt difficulty" as a catch-all. This is a defensible modeling choice, but a reader might ask: does "prompt difficulty" become so broad that it loses predictive content? If position effects, framing quality, relevant-to-irrelevant ratio, and compositional depth all collapse into "difficulty," the relational claim becomes near-tautological — effective context depends on everything about the prompt except its token count. The note could be clearer about whether "prompt difficulty" is a placeholder for multiple separable dimensions or a genuinely unified concept.

INFO:
- [Grounding alignment / vocabulary] The note says ConvexBench shows "performance collapsing with compositional depth at just 5,331 tokens." The ConvexBench source says 5,331 tokens is the count at depth 100. The source describes performance degrading "beginning as early as depth 5" and reaching F1 of approximately 0.2 at depth 100. The note's phrasing "at just 5,331 tokens" could be read as implying collapse happens at that specific token count, when actually the collapse is progressive across depths 5-100 and the 5,331 figure is the token count at the extreme end. The attribution is not wrong, but it compresses the progression into what reads like a point event.

- [Completeness / boundary case — simplest instance] Consider a trivial lookup task: "What is the capital of France?" in a context containing only that question. Effective context here is essentially model-independent and task-type-independent — every model succeeds, the task type is irrelevant, and difficulty is negligible. The note's framework handles this fine (the cost norm is very small, well under any threshold), but this simplest case reveals that the "relational" property only becomes interesting above some complexity floor. The note never identifies when the relational property starts to bite, which could help scope its practical applicability.

- [Completeness / boundary case — between enumerated items] The note draws a clean two-leg argument: Paulsen covers task type, ConvexBench covers complexity. But consider a case that sits between these: a long-context summarization task (Paulsen's domain) where the material has deeply nested argumentative structure (ConvexBench's domain). Does the note's framework predict what happens? The ||P||_t formalism suggests yes (the cost norm would be high on both dimensions), but neither source provides evidence for this joint case. This is an untested interior of the claimed space — not a flaw, but a gap the note could acknowledge more explicitly.

- [Internal consistency / definition drift] The note uses "complexity" in two related but distinguishable senses. In the opening paragraph: "one compositionally shallow and cleanly framed, the other requiring deep structured reasoning or burying the relevant information in a harder-to-use presentation" — here complexity includes both compositional depth and poor framing/presentation. In the ConvexBench paragraph, complexity is specifically compositional depth (nested function composition). Later, when connecting to the context-efficiency note's "two-axis model," complexity appears as the general "how hard the tokens are to use" dimension. These are compatible but not identical — compositional depth is one contributor to difficulty, not the whole of it. The note's caveats treat framing as part of "prompt difficulty," which partially resolves this, but the body switches between the narrow (ConvexBench-style compositional depth) and broad (general difficulty) senses without flagging the shift.

- [Grounding alignment / inference validity] The note claims its interpretation "is weaker and cleaner than treating MECW as a single parameterized scalar MECW(model, task_type, complexity)." The claim that relational framing is "weaker" than parametric framing is presented as self-evident, but it depends on what "weaker" means. A relational claim (X depends on Y and Z) is weaker in commitment than a parametric claim (X = f(Y, Z)) only if we mean "makes fewer quantitative predictions." But a relational claim that folds everything into a general cost norm ||P||_t without specifying what t captures could be criticized as weaker in a less flattering sense — it may be less falsifiable. The note does not address this tension.

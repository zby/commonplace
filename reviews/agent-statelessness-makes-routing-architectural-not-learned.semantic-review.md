=== SEMANTIC REVIEW: agent-statelessness-makes-routing-architectural-not-learned.md ===

Claims identified: 15

WARN:
- [Completeness] The five-level loading hierarchy (CLAUDE.md -> skill descriptions -> skill bodies -> type templates -> methodology notes) omits conversation history / chat context as a loading layer. In practice, the agent's context includes prior turns, tool outputs, and user-provided instructions within the session. These are not routed through the hierarchy but occupy context and influence behavior. The hierarchy claims to be "our concrete progressive disclosure system" but describes only pre-authored layers, not runtime-accumulated context. This is a boundary case that sits between the enumerated items.

- [Completeness] The degradation model claims a binary cliff: "KB-augmented -> generic." This understates the real degradation landscape. An agent with partial routing loaded (e.g., CLAUDE.md present but a specific skill body missing) degrades into a partially-augmented agent, not a fully generic one. The note acknowledges partial routing produces "confident execution with systematic errors" (Design consequences, item 3), which is a third degradation state — neither fully KB-augmented nor fully generic. The cliff metaphor and the "worse than no routing at all" observation are in tension: the cliff implies two states, but the partial-routing failure mode implies a continuum (albeit a steep, hard-to-navigate one).

- [Grounding] The note claims "The [methodology enforcement] gradient describes practices hardening from instruction -> skill -> hook -> script. The *practices* mature. The *agent* stays exactly as raw as it was on day one. This asymmetry is the core of the note: the system learns, the agent doesn't." The methodology-enforcement note actually describes a constraining gradient for enforcement reliability, not a learning process of the system. The source note explicitly frames it as "constraining" — reducing semantic underspecification and execution indeterminism. Calling this "the system learns" is a vocabulary mismatch: the source never uses "learns" or "learning" in this sense. The inference (constraining = a form of system learning) is plausible but is the note's own move, not something grounded in the source.

INFO:
- [Completeness] The three design consequences (engineering discipline, staleness detection, behavioral completeness) are presented as exhaustive: "Three requirements follow." A boundary case not clearly covered: **discoverability or searchability** — the requirement that routing infrastructure be findable by the agent in the first place. The note's progressive disclosure section implies this (the hierarchy must route correctly to the next layer), but the design consequences section doesn't name it as a separate requirement. It could be folded into "behavioral completeness," but that item focuses on edge-case reasoning and scope boundaries, not on the agent's ability to find the routing artifact in the first place. The fit is strained.

- [Completeness] The note's final paragraph acknowledges that "persistent memory across sessions would relax the statelessness assumption" but argues routing still scales with KB size. This is a hedged scope claim. A stronger boundary case: agents with tool-use capabilities (web search, file search) can discover knowledge not pre-routed to them. The note treats routing as the sole delivery mechanism, but search-based discovery is an alternative channel that partially relaxes the statelessness problem without persistent memory. The note's own KB (CLAUDE.md) includes search patterns as a routing mechanism, but the note's argument doesn't distinguish pre-authored routing from search-based discovery.

- [Internal consistency] The note says "The agent isn't a learner; it's a runtime. The KB isn't a curriculum; it's a deployment environment." But the human-LLM differences note (cited as motivation) explicitly warns against "naive mechanism-ism: treating the agent as 'just a text processor.'" The runtime/deployment metaphor is exactly the mechanistic framing that the upstream note cautions against. The note does parenthetically point to the human-LLM differences note as developing "why both pedagogical and purely mechanistic metaphors mislead," but then proceeds to use the mechanistic metaphor without qualification. The tension is acknowledged but not resolved.

- [Grounding] The note claims the instruction-specificity note says "match instruction specificity to loading frequency" and that this note explains *why* it is "architecturally necessary, not just convenient." The instruction-specificity note does state the principle, so the attribution is accurate. However, the instruction-specificity note already provides its own rationale (always-loaded context competes for attention) — it does not present itself as lacking a "why." The claim of providing the "deep rationale" is the note's own framing of the relationship, not something the source requests or implies is missing.

- [Completeness] The source/compiled distinction ("Methodology notes are source code. Skills, routing tables, type templates, and naming conventions are compiled artifacts") is a clean metaphor but omits a boundary case: CLAUDE.md itself. CLAUDE.md contains both routing (compiled) and methodology-level reasoning (source) in the same artifact. The Vocabulary section in CLAUDE.md, for example, provides definitions with reasoning context, not just imperative routing. CLAUDE.md doesn't map cleanly to either side of the binary.

PASS:

No pairwise contradictions found between sections beyond the tension noted above regarding the cliff metaphor.

- [Grounding] The context-efficiency note is cited as foundation for "why progressive disclosure exists." The attribution is accurate — the note correctly uses this as grounding for the progressive disclosure hierarchy.

- [Grounding] The indirection-cost note is cited as supporting "the reason lossy routing is dangerous." The source establishes that indirection costs context and interpretation overhead. The note's use of this to argue that routing omissions are dangerous (because the agent can't resolve missing information at runtime) is a valid inference from the source's cost model.

- [Internal consistency] The description ("Agents never develop navigation intuition... so all knowledge routing infrastructure is permanent architecture, not scaffolding that learners outgrow") faithfully represents the body's argument without eliding tensions.

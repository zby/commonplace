WARN:
- [Completeness — boundary cases for test 3] The three-test enumeration dismisses test 2 ("novelty relative to the model, not usefulness to the reader") as "well-defined but wrong." However, the note's own definition of reverse-compression in its final heuristic asks whether the note "make[s] structure accessible that wasn't before" for "an agent or human with access to this KB and general training knowledge." When the intended reader IS an LLM agent (the primary consumer per the KB's own framing), tests 2 and 3 substantially converge — what's derivable from training knowledge and what's already accessible to a model-reader are nearly the same thing. The note does not address this convergence, which weakens the claimed sharp distinction between tests 2 and 3. The gap matters most for the human-reader case, but the note does not restrict scope to human readers.

- [Grounding alignment — epiplexity scope] The note claims "Epiplexity — epistemic complexity extractable by a bounded observer — formalizes test 3." However, the epiplexity paper (ingest) defines epiplexity as learnable structural patterns extractable from sequential data under computational bounds — it measures structure in data distributions, not the broader notion of "usefulness to a reader" that test 3 describes. Test 3 asks whether the body "make[s] structure accessible to the *reader* that wasn't before," which includes factual content, surprising juxtapositions, and audience-specific relevance — dimensions the epiplexity formalism does not cover. The information-value-is-observer-relative note itself flags this gap in its open questions: "Observer-relativity applies to both patterns (require computational depth to extract) and facts (require prior knowledge to interpret). Are these the same phenomenon or two phenomena that share a surface shape?" The note treats epiplexity as fully formalizing test 3, but the formalization covers the pattern-extraction subset, not the full space of "adding information for a bounded observer."

INFO:
- [Completeness — boundary case: correct but unoriginal synthesis] The note defines reverse-compression as expanding "without adding extractable structure." Consider a note that correctly synthesizes two existing KB notes into a third, where the synthesis is logically derivable from the two sources but no reader had previously made the connection explicit. The connection is not "already accessible" in the sense that it requires traversal and inference to see it. Under the note's definition, this is ambiguous — the structure was technically accessible (the notes existed, the links could be followed) but the synthesis made it cheap to extract. The note's own qualifier about "surprising juxtapositions" partially handles this, but the line between "surprising connection" and "obvious connection" is left to reader judgment with no further guidance.

- [Completeness — boundary case: audience-dependent classification] The note acknowledges that reverse-compression is audience-relative ("structure was already accessible to its readers"). This means the same note could be reverse-compressed for one audience and genuinely informative for another. The note does not discuss how to handle this in practice — particularly for a KB that serves both agents (with training knowledge) and humans (with varying expertise). The validation heuristic names "the intended reader" as "an agent or human with access to this KB and general training knowledge," but this is a union of two very different observers, and what's already accessible to each differs substantially.

- [Internal consistency — link-removal test vs. network accumulation] The note argues that "the network is where epistemic complexity accumulates, not the prose of any single note" (claim 8), and then proposes a link-removal test: "remove the links — does the argument collapse, or does it read identically?" (claim 10). These two claims create a slight tension: if the network is where value accumulates, then a note whose argument collapses without links is dependent on the network but doesn't itself contribute accumulation — it's a pass-through. Conversely, a note that reads identically without links might still contribute its own epiplexity to the network as a destination. The test checks whether the note draws from the network, not whether it contributes to it. Both directions matter for resisting reverse-compression, but only one is tested.

- [Grounding alignment — link-contracts-framework] The note links to link-contracts-framework.md with the relationship "enables: articulated link relationships are what make links load-bearing rather than decorative." The link-contracts-framework note discusses link intent taxonomy, click-decision contracts, and automated testing — it addresses how to make links navigable and intentional, not directly whether links are "load-bearing" in the sense this note uses (i.e., whether removing them collapses the argument). The framework enables better link hygiene but does not directly address the semantic load-bearing property. The connection is reasonable but the "enables" relationship overstates the directness.

The attribution is accurate as a relationship between the two notes (the scope concern about epiplexity's coverage of the full claim is flagged separately as a WARN above, since it concerns the source paper, not the intermediate note).

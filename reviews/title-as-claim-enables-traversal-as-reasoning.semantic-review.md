WARN:
- [Completeness] The three-row taxonomy (single-claim / multi-claim / definitional) omits **procedural/instructional documents**. An instruction like "prose-review.md" is neither a single claim, a multi-claim framework, nor a definitional term-pinning note. Its title is imperative, not propositional or topical. It doesn't serve as a premise ("since [prose review]...") but also doesn't function like a reference spec. The note's own "shadow side" section mentions "procedural" ideas that resist claim titles, but the taxonomy table doesn't account for them — leaving a gap between the enumerated categories and the note's own acknowledgment.
- [Grounding alignment — domain coverage] The note claims that "a claim title is an affordance declaration" connecting to the linked note "instructions-are-typed-callables.md." However, the linked note discusses skills and instructions as typed callables with document type signatures — it is about **procedures that operate on documents**, not about how titles declare affordances. The linked note's concept of "affordance" refers to what operations a document type supports (verification, implementation, navigation), not what reasoning role a title plays in traversal. The note is making a metaphorical extension ("title as typed signature") and grounding it in a source whose domain (callable procedures with type signatures) is adjacent but distinct from the claim domain (titles as reasoning affordances).

INFO:
- [Completeness] The prior work enumeration lists four fields where propositional titles are established (academic writing, journalism, evergreen notes, Zettelkasten). A boundary case: **legal writing** and **patent claims** also use propositional/declarative titles systematically — patent claims are literally numbered propositions. This is not a gap per se (the note says "several fields," not "all fields"), and the note explicitly flags the survey as non-systematic. But legal drafting is particularly relevant given that this KB has a note on legal-drafting parallels to context engineering.
- [Completeness] The taxonomy distinguishes "argumentative" link semantics (since, because) from "referential" ones (see, as defined in), but there is an in-between case: **evaluative links** ("supports," "contradicts," "qualifies"). These are neither purely argumentative (they don't chain as premises in a syllogism) nor purely referential (they do express a stance). The linked note on link-contracts-framework lists intents like "Evidence" and "Rationale" that don't map cleanly to the argumentative/referential binary presented in the table.
- [Grounding alignment] The note says the Toulmin argumentation model "provides the formal structure behind the 'since/because' link semantics this enables." The Toulmin source describes six components (claim, grounds, warrant, qualifier, rebuttal, backing) as a framework for analyzing arguments. The note's mapping — claim titles are Toulmin claims, and "since"/"because" link semantics encode warrants — is a plausible analogical reading, but the Toulmin model is about the internal structure of a single argument, not about inter-document link traversal. The note is applying a within-argument framework to a between-document navigation pattern. The inference is reasonable but the fit is looser than "provides the formal structure" suggests.
- [Internal consistency] The note says "The title convention (claim vs topical) is independent of the type — any note can use a claim title." Two paragraphs earlier it says the type system maps onto this: "notes with claim titles may be promoted to structured-claim... spec, index, and other structural types carry topical titles." These two statements create a mild tension: if the convention is truly independent of type, then specs could carry claim titles too — but the note just said they carry topical titles. The resolution (specs typically carry topical titles but it's convention not requirement) is implicit rather than stated.

The claim that titles must carry signal for this to work is a reasonable inference from the loading hierarchy design — if the first layer is slim, the pointers (titles, descriptions) must be informative.

The three sections form a coherent argument arc without contradiction.

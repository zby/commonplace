=== SEMANTIC REVIEW: minimum-viable-vocabulary-is-the-set-of-names-that-maximally-reduces-extraction-cost-for-a-bounded-observer.md ===

Claims identified: 14

WARN:
- [Completeness] The note claims "Two mechanisms already in the KB explain why a set of names can have this effect — why vocabulary, specifically, is the right unit of extraction cost reduction." This is a scope claim (exactly two, and they are sufficient). A third mechanism is plausible and unaddressed: **chunking / categorical perception**. Cognitive science shows that having a name for something changes perceptual and memory processing — named categories are processed faster and grouped more efficiently (categorical perception, verbal overshadowing literature). This is distinct from both "naming amortizes discovery cost" (which is about recognition of instances) and "distillation targets a bounded observer" (which is about compression for a use case). Chunking explains why a vocabulary changes the observer's computational capacity itself, not just what they can recognize or what was compressed for them. The "two mechanisms" framing may be undercounting.
- [Grounding — domain coverage] The note's central claim is about "extraction cost for a bounded observer," grounded via the information-value note and the epiplexity paper. However, the epiplexity framework (Finzi et al.) measures extractable *structure* (learnable patterns in sequential data for a computationally bounded model), not extractable *information* in the broader sense. The note's MVV concept covers both structured patterns and factual/terminological knowledge (e.g., knowing what "BRDF" means in 3D graphics is a fact, not a learnable pattern). The information-value note itself flags this gap in its Open Questions: "Observer-relativity applies to both patterns (require computational depth to extract) and facts (require prior knowledge to interpret). Are these the same phenomenon or two phenomena that share a surface shape?" The epiplexity framework grounds the pattern-extraction half of the MVV claim but does not cover the factual-knowledge half. The note presents epiplexity's prequential coding as a candidate measure for evaluating MVV without flagging this scope mismatch.
- [Completeness — boundary case] The **simplest possible instance** of MVV — a single term — raises a question the note doesn't address. If the domain is trivially small (say, a domain with one core concept), the "optimization" framing collapses: there is nothing to optimize over, and the MVV is just "learn the term." The note's optimization framing ("maximally reduces extraction cost") implicitly assumes a domain rich enough that term selection is non-trivial, but this assumption is unstated. More importantly, the optimization claim requires a cost function and a search space, but the note provides neither formally — it gestures at epiplexity but acknowledges no measure exists for the actual MVV evaluation. The claim is "testable in principle" but the gap between principle and practice is wider than the note suggests.

INFO:
- [Completeness — boundary case] The note distinguishes between human and agent MVV in the Open Questions ("humans retain vocabulary across sessions; agents need it injected each session") but the body's two-mechanism framework does not address this distinction. The distillation mechanism describes MVV as targeting "a newcomer" with "minimal context budget," but for agents, the context budget is a hard token limit, whereas for humans it is a soft cognitive limit. These are different optimization constraints that could yield different optimal vocabularies for reasons beyond "different computational limits" — e.g., an agent can hold a longer flat list in context but may lack the associative retrieval a human uses to activate a term at the right moment. The Open Questions acknowledge this but the body's framing treats the distinction as simpler than it may be.
- [Grounding — vocabulary vs. ontology] The note states it uses "vocabulary" rather than Kim's "ontology" because "the claim is about names that reduce extraction cost, not about a formal ontological commitment." This is a reasonable terminological choice, but it slightly misrepresents the distance from Kim. Kim's tweet thread does not use "ontology" in the formal philosophical sense either — it uses it loosely as "a set of key terms." The note implies a sharper distinction between its framing and Kim's than may actually exist. The real difference is the optimization framing (which terms? how many?), not the ontology-vs-vocabulary word choice.
- [Internal consistency — definition drift] The note defines MVV as "the smallest set of names such that, once in context, the observer can extract domain structure that was previously inaccessible" (paragraph 3). The very next sentence restates this as "the vocabulary that maximally reduces extraction cost for a given observer." These are not identical: the first definition is about a binary threshold (inaccessible → accessible), while the second is about a continuous optimization (maximally reduces cost). A set that makes structure accessible is not necessarily the set that maximally reduces cost — you could have a larger set that reduces cost further by making extraction faster, not just possible. The note treats these as synonymous without acknowledging the gap.
- [Grounding — testability claim] The note claims the framing "makes the concept testable in principle" and names epiplexity's prequential coding as a candidate measure. The epiplexity paper's prequential coding measures area under the loss curve above final loss for a model learning sequential data. Applying this to MVV evaluation would require: (a) defining a learning task over domain artifacts, (b) comparing model performance with vs. without vocabulary in context, and (c) treating the difference as "extraction cost reduction." This is a plausible but non-trivial operationalization that the note does not sketch — the testability claim is aspirational rather than concrete.

PASS:
- [Grounding — Kim source] The note accurately represents Kim's tweet thread. Kim does propose "minimum viable ontology," does frame terms as "conceptual thresholds," and the prototype (domainmaps.co) is correctly described as providing neither evaluation measure nor fixed observer. The ingest file corroborates all attributions.
- [Grounding — discovery note] The note claims "Discovery argues that the highest-value act in a knowledge system is creating a named concept — once a structure has a name, recognizing further instances becomes cheap." The discovery note does say exactly this: "once a structure has a name, recognizing new instances becomes cheap. The naming amortizes the discovery cost." Attribution is accurate.
- [Grounding — distillation note] The note claims MVV is "a distillation of domain knowledge where the target is a newcomer and the context budget is minimal." The distillation note defines distillation as "compressing knowledge so that a consumer can act on it within bounded context" and explicitly states "Different operational contexts need different extractions from the same source, so multiple distillations are normal." The MVV-as-distillation framing is consistent with the source.
- [Grounding — information-value note] The note claims that "a restructuring that preserves or even reduces Shannon entropy can still make structure accessible to a bounded observer that could not previously extract it." The information-value note makes this exact argument, grounded in the epiplexity paper's "Paradox 1" (deterministic transformations cannot increase classical information yet do increase extractable structure). Attribution is accurate.
- [Internal consistency — overall] The note's argument flows coherently: it introduces the problem (extraction gap), cites the source (Kim), reframes via the KB's framework (information-value), identifies two mechanisms (naming, distillation), synthesizes them into an optimization claim, and addresses testability. No section contradicts another. The definition drift noted in INFO above is a tension but not a contradiction.

Overall: 3 warnings, 4 info

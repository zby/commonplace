=== SEMANTIC REVIEW: design-methodology-borrow-widely-filter-by-first-principles.md ===

Claims identified: 12

WARN:
- [Completeness] The adoption filter enumerates four tiers (first principles, programming patterns fast pass, empirical observation, everything else) but the boundary between "everything else earns its way in" (tier 4) and "first principles reasoning is the main filter" (tier 1) is unclear. Tier 4 says "Cognitive science patterns, HCI patterns — these get adopted when first-principles reasoning supports them." But tier 1 already says first principles is the main filter for everything. The tier 4 category collapses into tier 1 — it is not a distinct adoption path but a restatement that non-programming, non-observation sources must pass the default filter. The four-tier presentation implies four distinct mechanisms, but there are really three: first-principles derivation, programming fast pass (reduced bar), and empirical observation (different evidence type). The "everything else" tier is not an independent category.

- [Completeness] The note lists four first principles: "finite context windows, no import/resolution mechanism, agents reason over text, everything loaded must compete for attention." The last two overlap significantly — "agents reason over text" and "everything loaded must compete for attention" are both consequences of the text-in-text-out bounded context architecture. A boundary case: the principle "agents have no persistent state between sessions" is used later in the "Why the asymmetry" section ("no persistent state between sessions") but is not listed among the first principles in the adoption filter section. If statelessness is load-bearing enough to explain why cognitive science analogies fail (the asymmetry argument), it should be in the enumerated first principles. Its absence from the list but presence in the argument is an inconsistency.

- [Grounding — domain coverage] The note claims "Convergence across independent projects is stronger evidence than any single design argument" and cites Thalo as the evidence. But the Thalo review note describes convergence on *programming patterns applied to knowledge management*, not convergence on the specific claim that programming patterns deserve a fast pass over cognitive science patterns. Thalo built a compiler for knowledge management — that validates the programming-patterns-transfer hypothesis. It does not validate the asymmetry claim (that programming should get a fast pass while cognitive science should not). Thalo's convergence evidence covers one part of the note's argument (programming transfers) but is presented as if it supports the broader methodological hierarchy.

INFO:
- [Completeness] Boundary case: mathematical/formal reasoning as a source discipline. Mathematics is formal and compositional (like programming) but operates on abstract structures rather than text-based systems. Would mathematical patterns (e.g., category theory for composability, information theory for context budgets) get the programming fast pass or the "everything else" treatment? The note's criterion is "structurally similar to interpreters interpreting programming languages" — mathematics is structurally similar to programming but not specifically about text processing. This edge case is not covered by the current framework.

- [Completeness] Boundary case: design patterns from other agent/AI systems (e.g., robotics, game AI, classical planning). These are programming-adjacent and deal with bounded processors under constraints, but the constraints are different (physical environment vs. text context). The note's fast-pass criterion ("bounded processors composing text under constraints") would exclude robotics patterns since they don't involve text, but the deeper structural parallel (bounded processor, finite state, compositional actions) might justify a fast pass. The framework does not address programming-adjacent disciplines that share some but not all structural properties.

- [Grounding] The note states legal drafting "operates in the same medium as prompts: natural language with irreducible ambiguity" and that "we haven't yet borrowed a concrete technique from law and applied it successfully." The linked legal drafting note, however, documents a detailed table of six legal techniques with context engineering analogues (defined terms, structural conventions, enumeration, canons of interpretation, precedent, codification) and notes that ABC (Agent Behavioral Contracts) "reinvents legal enforcement patterns without citing law as a source discipline." The legal drafting note itself seems to describe successful mappings. The methodology note's claim that law is entirely "untested" may be slightly stronger than warranted — the legal note shows the mappings exist conceptually even if they haven't been explicitly borrowed-and-applied in practice. The distinction between "we identified the mapping" and "we borrowed and applied the technique" is doing important work here but is not made explicit.

- [Internal consistency] The note says "Empirical observation is the second strongest source — but weaker than first principles" and positions it below the programming fast pass in the hierarchy. But the text also says empirical observation "doesn't go through the borrowing/adoption filter at all. It's evidence from *this* system, not transferred from another domain." If empirical observation bypasses the adoption filter entirely, it is not really in the same hierarchy as the other tiers — it is a different kind of evidence, not a weaker version of the same kind. Ranking it as "second strongest" in a hierarchy it explicitly sits outside of creates a mild tension. The note partially resolves this by explaining the quantity-vs-weight distinction, but the initial framing ("second strongest source") invites comparison within a single scale that the subsequent text undermines.

- [Completeness] Boundary case: what happens when first principles conflict? The note presents first principles as the strongest filter, but the listed principles could tension against each other (e.g., "agents reason over text" might push toward verbose explanatory context, while "everything loaded must compete for attention" pushes toward compression). The methodology for resolving conflicts between first principles is not addressed.

PASS:
- [Grounding] The claim that Thalo "independently arrived at building an actual compiler for knowledge management — Tree-Sitter grammar, typed entities, 27 validation rules" is accurately attributed. The Thalo review note confirms all three specifics (Tree-Sitter grammar, typed entities, 27 validation rules) and describes the system as independently developed.

- [Grounding] The characterization of Arscontexta — "249 research claims grounded in cognitive psychology are acknowledged but not adopted wholesale" and the spreading activation concern — aligns with the Arscontexta review note, which describes 249 interconnected claims and explicitly questions whether the spreading activation model transfers across the scale difference between neural and note-graph systems.

- [Grounding] The programming-practices-apply-to-prompting note supports the claim that programming patterns transfer because "both domains solve the same problems: making behaviour predictable, making systems composable, making artifacts verifiable." The vocabulary and framing are consistent between the two notes.

- [Internal consistency] The note's core structure — the adoption filter (what gets in) followed by the asymmetry argument (why programming gets privileged treatment) — is internally coherent. The asymmetry section provides the theoretical justification for the fast pass described in the adoption filter. The two sections reinforce rather than contradict each other.

- [Internal consistency] The distinction between first principles (scarce, individually strong) and empirical observation (plentiful, individually weak) is consistently maintained throughout the note. The wikiwiki principle reference at the end ("capture observations freely, then refine") is consistent with the quantity-vs-weight framing.

Overall: 3 warnings, 4 info
